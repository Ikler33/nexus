# E2E web-смоук (P0-3)

Playwright-смоук **браузерной сборки** приложения — единственный межкомпонентный авто-оракул
фронта (проводка App ↔ ActivityBar ↔ main-вью ↔ оверлеи ↔ редактор ↔ чат ↔ агент). Это
behavior-preserving гарантия фронт-рефакторинга (REFACTOR-PLAN §2): юниты сторов не ловят
слоение вьюх, Esc-семантику и мёртвые кнопки.

## Что доказывает / что НЕ доказывает (честность, W-22)

**Доказывает:** прод-сборка Vite (`vite preview`, та же сборка, что идёт в бандл) под движком
**webkit** (ближайший к WKWebView Tauri) корректно проводит пользовательские сценарии против
мок-бэкенда `src/lib/mock/*` (мок обязан зеркалить контракт Rust-команд — см.
`feedback: mock must match backend` и авто-гейт паритета).

**НЕ доказывает:** поведение **shipped Tauri-app** — Rust-команды, файловую систему, окно,
меню, реальные LLM/git. Зелёный смоук ≠ «фича готова в приложении»; DoD остаётся
«проверено в живом app». Имя CI-джобы говорит это явно: `E2E web smoke (web+mock-слой,
НЕ shipped-app)`.

## Как гонять локально

```bash
pnpm e2e            # из корня репо: сборка + vite preview :4173 + прогон (webkit)
```

- Конфиг: `apps/desktop/e2e/playwright.config.ts`. Сервер поднимает webServer-секция
  (порт **4173**; 1420 — дев-стенд, не занимать).
- Разовая установка движка: `npx playwright install webkit` (в CI — `--with-deps`).
- Отчёт при падении: `apps/desktop/e2e/playwright-report/`, артефакты (скриншоты/trace) —
  `apps/desktop/e2e/test-results/` (оба в .gitignore).
- Анти-флейк-политика: ретраев нет; перед мержем изменений смоука — два зелёных прогона
  подряд. Sleep-магия запрещена — только web-first assertions.

## console-гейт

Каждый тест валится при `console.error`/`pageerror` (фикстура `fixtures.ts`). Whitelist
допустимых ошибок заведён **пустым**; расширять только с комментарием-обоснованием.

## Правило поддержки (реестр коннектора, будущий F-8)

**Новая main-вью или оверлей → добавь её в смоук в том же PR:**

- main-вью → `MAIN_VIEWS`/`mainViewAnchor` в `fixtures.ts` (nav-смоук подхватит попарные
  переключения автоматически) ;
- оверлей/панель → тест в `overlays.spec.ts`: путь открытия из UI, видимый якорь,
  Esc-семантика КАК ЕСТЬ (focus-trap → Esc закрывает; без трапа — явная кнопка).

Сценарии со `test.skip` + `TODO(P0-2)` ждут мержа ветки `test/p0-2-mock-parity`
(триггеры exec/report/демо-ошибка в моках) — снять skip при ребейзе.
