# Workspace: группы/вкладки/буферы (`src/stores/workspace.ts`, `components/workspace`)

> Срез Ф0-9 (§4.1, **Б12**). Модель групп/вкладок вместо одиночного `currentFile` (ретрофит =
> переписывание ядра). Активный документ = активная вкладка активной группы → контекст
> AI/backlinks/suggest. Критерии **AC-Б12-1/2**.

## Стор
- `Buffer { path, doc, dirty }` — один на путь (общий между группами).
- `EditorGroup { id, tabs[], activeTab }`; массив `groups`, `activeGroupId`.
- Действия: `openFile(path, group?)`, `openLink`, `setActiveTab`/`setActiveGroup`, `closeTab`
  (GC буферов без ссылок, всегда ≥1 группа), `splitRight` (новая группа с активной вкладкой),
  `updateBufferDoc` (правка → dirty), `saveBuffer`, `reset`.
- Селекторы: `activeBuffer(s)`, `activePath(s)`.

## UI
- `EditorArea`: группы-сплиты в ряд; пусто → подсказка.
- `GroupPane`: панель вкладок (active/dirty/закрыть) + split-кнопка + `Editor` (key=groupId) +
  `BacklinksBar({path})`. Клик по панели → активная группа. Контракт CM6 §4.1: один view на
  группу, смена вкладки → `dispatch` (без пересоздания).

## Инварианты
- Правки/`dirty` буфера сохраняются при переключении вкладок — **AC-Б12-2**.
- Backlinks/контекст — из активной вкладки СВОЕЙ группы; глобальный активный буфер = активная
  вкладка активной группы — **AC-Б12-1**.
- Открытие vault сбрасывает workspace (`openVaultFlow`).

## Тесты
- Стор: открытие/вкладки/переключение, dirty при переключении (Б12-2), splitRight + контекст
  активной группы (Б12-1), save, closeTab + GC, openLink.
- Компоненты: FileTree открывает файл в активной группе.
