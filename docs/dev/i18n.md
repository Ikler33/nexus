# i18n RU/EN (`src/i18n`)

> Срез Ф0-10 (§9). i18next + react-i18next. Критерии **AC-I18N-1…5**.

## Setup (`setup.ts`)
- Ресурсы ru/en (namespace `translation`). `detectLocale`: сохранённый выбор → `navigator.language`
  → `en` (С-11 — новый русскоязычный пользователь сразу видит ru).
- `changeLocale(locale)`: сохраняет в localStorage + `i18n.changeLanguage` (состояние сторов не теряется).
- Встроенные форматтеры i18next (Intl): `{{count, number}}`.

## Форматирование (`format.ts`)
- `formatNumber(n, locale)` — `Intl.NumberFormat` (50000 → «50 000» / «50,000»).
- `compareEntries` / `collatorFor` — `Intl.Collator` (numeric, base): каталоги выше файлов,
  кириллица сортируется корректно.

## Использование
- Компоненты — `useTranslation()` / `t()`. Все строки = ключи (`ru.json`/`en.json`), DESIGN §9.
- Плюралы: `backlinks.count_one/_few/_many` (ru), `_one/_other` (en) — `t('backlinks.count', {count})`.
- Команды — `titleKey`; палитра рендерит `t(titleKey)`.
- Дерево/результаты сортируются `compareEntries` (Collator активной локали).
- Переключатель языка — в шапке sidebar (иконка Languages).

## Тесты
- ru/en одинаковый набор ключей (без плюрал-суффиксов) — **AC-I18N-1**.
- ru-плюралы one/few/many — **AC-I18N-2**; Intl-числа — **AC-I18N-3**; Collator-сортировка —
  **AC-I18N-4**; детекция + смена языка — **AC-I18N-5**. Тест-окружение детерминировано в ru.

## Плагинные namespace (AC-I18N-7, Ф2-3) ✓
Плагин шлёт `ui.addTranslations {локаль→{ключ→строка}}` (см. `plugins.md`); фронт-релей кладёт строки в
i18next namespace **`plugin`** вложенно: `{<dir>:{<key>:value}}` → итоговый ключ `plugin:<dir>:<key>`
(i18next режет ключ по `:` на ns + вложенный путь, поэтому именно вложенно, не плоской строкой). Команда
плагина (`registerCommand` с `titleKey`) получает `titleKey = plugin:<dir>:<key>` — палитра резолвит его
через `t()` и перерисовывается при смене языка. Проверено в превью (EN↔RU меняет заголовок команды).
