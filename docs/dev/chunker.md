# Markdown-чанкер (`src-tauri/src/chunker`)

> Срез Ф1-2 (§6.1). Готовит чанки для RAG-эмбеддинга. Критерий **AC-Б4-1** (эмбеддинг по чанкам).

## Стратегия
- **Frontmatter вырезан** (`parser::split_frontmatter`) — в чанки идёт только тело; `char_start/end`
  считаются в ИСХОДНОМ файле (для dedup overlap §6.7 и jump-to-section).
- **Разбиение по ATX-заголовкам** → секции с `heading_path` (`"H1 > H2"`). Заголовки внутри
  fenced-code игнорируются.
- Каждая секция: единицы = СЛОВО прозы / цельный fenced-code (атомарно). **Sliding window**
  пакует единицы ≤ `max_tokens`; начало следующего окна сдвигается назад на `overlap_tokens`
  (overlap ВНУТРИ окна, не «+64 сверх лимита» — иначе чанк > бюджета §6.4). Код не рвётся;
  единица крупнее лимита → отдельное окно.
- `token_count` — по финальному тексту чанка.

## Tokenizer
Трейт `Tokenizer`; пока `WordTokenizer` (число слов) — **placeholder**. Реальный токенайзер
эмбеддера — Ф1-3 (для кириллицы посимвольная эвристика врёт 1.5–2×); точные границы уточнятся с ним.

## Chunk
`{ chunk_index, content, char_start, char_end, heading_path, token_count }` — 1:1 к таблице `chunks`.

## Тесты
короткий → 1 чанк (heading_path None); frontmatter вырезан + смещения в исходный файл; заголовки →
`heading_path`; длинная секция → несколько чанков с overlap (≤ max_tokens); fenced-code атомарен.
