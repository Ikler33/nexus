# Eval-харнесс качества RAG (`src-tauri/src/eval` + `eval/`)

> Срез Ф1-10 (§6.6, **AC-EVAL-1..6**). По образцу `sa-eval`: golden `вопрос→файлы`, метрики
> **recall@8 / nDCG@8 / MRR**, регресс-гейт против зафиксированного baseline. «Correctness over speed»
> неисполним без измерения — этот харнесс и есть измерение; он же разблокирует калибровку порогов/реранкер.

## Данные
- **`eval/golden.json`** (AC-EVAL-1): корпус разнотемных заметок (RU/EN) + кейсы `query → relevant[]`,
  включая **кросс-язычные** (AC-EVAL-6: RU-запрос по EN-заметке и наоборот). Версионируется, расширяется.
- **`eval/baseline.json`** (AC-EVAL-2): пороги `recall_at_k/ndcg_at_k/mrr` + **условия** (модель, сервер,
  k, набор — AC-EVAL-4). Зафиксированы по фактическому прогону Фазы 1; обновлять осознанно при улучшении.

## Код (`eval/mod.rs`)
- Метрики (бинарная релевантность на уровне ФАЙЛОВ; выдача чанков схлопывается в файлы):
  `recall_at_k`, `reciprocal_rank`, `ndcg_at_k` — чистые, юнит-тестируемы.
- `run_eval(reader, vectors, embedder, cases, k) -> EvalReport`: на каждый кейс — `hybrid_search` →
  уникальные пути → метрики; агрегат = среднее. `EvalReport`/`CaseResult` (Serialize, для отчёта).
- `index_corpus(root, docs, embedder, vectors)`: строит temp-vault из корпуса и индексирует (RAG).
- `load_golden` / `load_baseline` (`include_str!`).

## Прогон (AC-EVAL-3 гейт)
```
# нужен embedding-сервер :8081 (условия из baseline.json)
cargo test eval::tests::live_eval_meets_baseline -- --ignored --nocapture
```
Печатает отчёт (условия + recall@8/nDCG@8/MRR + по-кейсный разбор с hits) и **падает при метриках ниже
baseline** (регресс-гейт). Любое изменение чанкера/RRF-весов/порогов/модели → перепрогнать (AC-EVAL-3).

## Фактический baseline (Фаза 1, nomic-embed-text @ :8081, k=8, 10 кейсов)
**recall@8 = nDCG@8 = MRR = 0.800.** 8/10 кейсов идеальны (релевантный на ранге 1). **2 промаха —
кросс-язычные** (EN→RU `budget`, RU→EN `photosynthesis`): nomic англоцентричен → cross-lingual проседает.
Это **количественно подтверждает риск ADR-005**: AC-EVAL-6 закроется мультиязычным эмбеддером (bge-m3/e5),
тогда baseline поднимаем. Калибровка решений «RRF / порог suggest / 8 чанков» — на этом наборе, не с потолка.

## Тесты
- Юнит (CI): математика метрик (recall/MRR/nDCG), парс golden+baseline, сквозная проводка `run_eval` на
  mock (точное слово → файл в top-k → recall=1). **Живой** (`#[ignore]`): прогон nomic ≥ baseline.
