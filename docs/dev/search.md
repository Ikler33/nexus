# Поиск (`src-tauri/src/search` + сайдбар)

> Ф0-7 — по **метаданным** (title/path/tags). Ф1-6 — **гибридный по ТЕЛУ** (вектор + FTS5 → RRF, §6.2).

## Метаданные (Ф0-7)
- `search_notes(reader, query) -> Vec<NoteRef>`: `files` LEFT JOIN `file_tags`/`tags`,
  `path/title/tag LIKE %q%` (спецсимволы LIKE экранируются, `ESCAPE '\'`), `LIMIT 100`. Команда `search_vault`.

## Гибридный поиск по телу (Ф1-6, §6.2)
`hybrid_search(reader, vectors?, embedder?, query, limit) -> Vec<SearchHit>`. Две независимые выдачи
кандидатов (по `CANDIDATES=50` каждая), затем слияние:
- **Вектор** (семантика): `embedder.embed_query(q)` → `VectorIndex::search` → ранжированные `chunk_id`.
  Эмбеддинг запроса — ВНЕ блокировки read-пула (лок снят в команде до сетевого вызова).
- **FTS5/BM25** (лексика): `fts_chunks MATCH … ORDER BY rank`. Запрос санитизируется `fts_query`:
  токены по не-буквенно-цифровым границам (юникод → кириллица цела), в кавычках, через `OR` — спецсинтаксис
  FTS не утекает (нет инъекции в MATCH).
- **RRF** (`rrf_fuse`, §6.2): score = `Σ 1/(k+rank)` (k=`RRF_K`=60, rank 1-based) по спискам, где id встретился;
  сорт по score↓, тай-брейк `chunk_id↑`. Сливаем РАНГИ, а не «сырые» score (cos и BM25 — разные шкалы).
- Топ-`limit` → один запрос резолва метаданных+содержимого (`chunks JOIN files`, `IN`-список) → `SearchHit`
  (path/title/heading_path/snippet/score). Сниппет — схлопнутые пробелы, обрез по символам до `SNIPPET_CHARS`.

**Изящная деградация:** нет эмбеддера → только FTS-ветвь; запрос пуст / обе ветви пусты → пусто.
Команда `search_content(query, limit?)` (`limit` дефолт 10, потолок 50) тянет `vectors`/`embedder` из
`VaultContext`. Контракт фронта: `tauriApi.search.searchContent` + мок `mock/vault.ts`.

## Фронт
- `Sidebar`: поле поиска по метаданным (debounce 150 мс) + дерево/результаты. Вне Tauri — мок.
- Поиск/чат по содержимому (UI поверх `searchContent`) — Ф1-8.

## Ограничения / дальше
- **Граф — 3-й ранг в RRF** (§6.2, REVIEW С-4): `rrf_fuse` уже N-арный, граф-близость к центру-файлу
  добавится третьим списком (а НЕ аддитивным `+0.2`) там, где есть центр (чат-RAG Ф1-7+, suggest Ф1-9).
- **AC-Б6-2**: префильтр по метаданным (папка/тег/дата) ДО KNN — когда появится UI фильтров.
- **Dedup overlap-чанков** (§6.1/§6.2): соседние перекрывающиеся чанки одного файла схлопывать в выдаче.
- Сниппет — обрез чанка без подсветки; FTS5 `snippet()`/`highlight` — refinement.
- Реранкер (jina, :8082) поверх топ-N гибрида — позже (ADR-005, опционально).

## Тесты
- Rust (метаданные): path/title/tag, пустой/пробельный запрос.
- Rust (гибрид): `rrf_fuse` (общий в обоих списках ранжируется выше), `fts_query` (токенизация/кавычки/OR),
  FTS-only выдача (без эмбеддера), сортировка по score + резолв метаданных, пустой/без-совпадений запрос.
  **Живой** (`#[ignore]`, nomic :8081): запрос без лексических пересечений → семантический топ из вектора. ✓
- Фронт: дерево↔результаты (метаданные); мок `searchContent` (score↓, limit, пустые случаи).
