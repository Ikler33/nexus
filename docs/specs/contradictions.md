# Vision→AC: Поиск противоречий

> Перевод vision-фичи «Поиск противоречий в базе» (BACKLOG §Идеи) в реализуемую спеку. Фоновая LLM-фича
> поверх планировщика (ADR-007) и RAG-эмбеддингов (bge-m3). Находит **конфликтующие / устаревшие**
> заметки: пары-кандидаты по семантической близости → LLM судит, есть ли противоречие.

## 1. Продуктовые решения (зафиксированы владельцем)

| ID | Решение | Значение |
|----|---------|----------|
| **D1** | Триггер | **Вручную (кнопка) + по расписанию** (фон, run-if-overdue; backpressure S5) ✅ |
| **D2** | Где показывать | **Отдельная панель** (как «Дайджест»/«Цели»: иконка в титлбаре + модалка) ✅ |
| **D3** | Типы противоречий | **hard / soft / temporal** (жёсткое фактическое · мягкое по тону/выводу · временное — устарело) — рекоменд. дефолт из BACKLOG |
| **D4** | Отбор кандидатов | пары заметок с **косинус-близостью ≥ порога** (одна тема) по bge-m3 (usearch); только **топ-N пар** за прогон (стоимость) |
| **D5** | Стоимость/кэш | лимит LLM-судейств за прогон (`MAX_JUDGE`); прогон **заменяет** прошлый результат; кэш по контент-хэшу пар — **отложен** (CT-3, no silent caps) |

## 2. Скоуп v1 / non-goals

**v1:** пары-кандидаты (max-sim из готового `get_related_notes`) → LLM-судья (JSON-вердикт hard/soft/temporal
+ объяснение) → таблица `contradictions` → панель (список, клик открывает заметку) + кнопка «Найти сейчас» +
фоновый run-if-overdue. Анти-инъекция (AC-SEC-7): тексты заметок — данные в маркерах.

**non-goals v1:** кэш по mtime/хэшу (CT-3); «разрешить/объединить/архив» действия (только показ); попарный
обход ВСЕГО vault без лимита (дорого — берём топ-N пар); ранжирование важности противоречий.

## 3. Acceptance Criteria (тестируем механику через мок-LLM, НЕ качество)

- **AC-CT-1 (kind+таблица)** kind `contradictions` собирает пары-кандидаты, судит мок-LLM, пишет
  найденные в `contradictions` (path_a, path_b, type, explanation). Пустой/без векторов vault → без записи.
- **AC-CT-2 (отбор пар)** кандидаты — пары с близостью ≥ `SIM_THRESHOLD`, дедуп (a<b), топ-`MAX_PAIRS`.
- **AC-CT-3 (вердикт)** ответ LLM парсится в `{contradiction, type, explanation}`; `contradiction=false` →
  не пишем; невалидный/`true` без типа → дефолт-тип `soft`, не падаем (устойчивый парс, в т.ч. ```-фенсы).
- **AC-CT-4 (замена)** новый прогон заменяет прошлые результаты (не плодит дубли).
- **AC-CT-5 (backpressure)** kind уступает интерактиву (S5: `defer_under_interactive=true`).
- **AC-CT-6 (триггер)** команда `generate_contradictions` (вручную, требует chat+векторы) + run-if-overdue
  seed на открытии; дедуп активной джобы (нет пары одновременных).
- **AC-CT-7 (панель)** список противоречий (две заметки + тип + объяснение), клик открывает заметку;
  пустое состояние; кнопка «Найти сейчас»; refetch по `jobs:changed`.

**НЕ тестируем (human-eval):** качество/релевантность вердиктов LLM — вероятностный вывод.

## 4. Нарезка
1. **CT-1 Бэкенд:** миграция `contradictions`; модуль (candidate_pairs · judge-промпт · parse · handler ·
   list) + команды; Rust-тесты (FakeChat).
2. **CT-2 UI:** панель из титлбара + стор + `tauriApi.contradictions.*` + i18n + refetch по `jobs:changed`.
3. **CT-3 ✅ кэш** по хэшу сниппетов пар (`contradiction_cache`, миграция 007 — не пере-судим
   неизменённое). Отложено: **действия** «разрешить/объединить/архив» + GC стейл-кэша (BACKLOG).

## 5. Зависимости
- **Планировщик (ADR-007)** + **backpressure (S5)** — ✅. **bge-m3 эмбеддинги/usearch** — ✅ (`get_related_notes`).
- **Chat (ADR-005)** — судья. JSON-вердикт: Gemma 4 даёт чистый JSON (memory eval); парс всё равно
  устойчив к ```-фенсам.
