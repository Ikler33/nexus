# Спека: Персистентная память агента (MEM)

> Vision-фича из `docs/BACKLOG.md` §«Идеи / vision». Сессия «vision→AC» проведена с владельцем
> 2026-06-15 — решения D1–D6 ниже зафиксированы. Это **слой ЯВНЫХ ФАКТОВ** о пользователе/проектах
> («пишу на Rust», «дедлайн проекта X — пятница»), отдельный от уже готового RAG-по-переписке (N4b,
> `chat_log::search_memory` + `chat_vectors`). MEM хранит дискретные факты, инжектит их в контекст ответа.

## Назначение
ИИ-напарник помнит факты о тебе и проектах между сессиями и подмешивает их в ответ — «второй мозг»
знает контекст без повторов. Не дублирует N4b (тот достаёт релевантные ПРОШЛЫЕ СООБЩЕНИЯ); MEM — это
курируемый набор ФАКТОВ-утверждений, которыми управляет пользователь.

## Зафиксированные решения (vision→AC, 2026-06-15)
- **D1 — Источник фактов:** ЯВНАЯ команда/кнопка («в память») + ПОДТВЕРЖДЁННОЕ авто. Авто = быстрая
  модель ПРЕДЛАГАЕТ факт после обмена («Запомнить: …? ✓/✗») — НЕ пишет сама. Ноль молчаливых записей.
- **D2 — Инжекция:** факты-ПИНЫ («всегда в контексте») + top-k семантически близких текущему вопросу
  (по эмбеддингу). Блок памяти — ДАННЫЕ в injection-маркерах (анти-инъекция, как N4b/digest).
- **D3 — Хранение:** таблица БД `memory_facts` (миграция 017) + индекс эмбеддингов `memory_vectors.usearch`
  (ключи = id факта, dim эмбеддера). Сниппет не храним отдельно — текст факта и есть вход эмбеддинга.
- **D4 — Контроль:** модалка «Память ИИ» (как Goals/Digest): список фактов, пин/анпин, редактирование,
  удаление, ручное добавление. Полная прозрачность — доверие к «второму мозгу».
- **D5 — Дефолт:** фича ВЫКЛ по умолчанию (`aiAgentMemory`, как `aiChatMemory`) — приватность-first;
  включается в Настройках → AI.
- **D6 — Кап/вытеснение:** мягкий кап `MEM_CAP` фактов. При переполнении — НЕ авто-эвикция: панель
  подсвечивает старые/редко-используемые для ручной чистки. Пины не считаются к капу и не вытесняются.

## Модель данных (миграция 017)
```sql
CREATE TABLE memory_facts (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    text        TEXT    NOT NULL,           -- сам факт (вход эмбеддинга)
    pinned      INTEGER NOT NULL DEFAULT 0, -- 1 = всегда в контексте, не вытесняется (D2/D6)
    source      TEXT    NOT NULL,           -- 'explicit' | 'auto' (D1)
    created_at  INTEGER NOT NULL,           -- unix-секунды добавления
    used_at     INTEGER NOT NULL DEFAULT 0  -- последний раз подмешан в контекст (для подсветки старых, D6)
);
```
Индекс эмбеддингов: `memory_vectors.usearch` (зеркало `chat_vectors`), ключ = `memory_facts.id`.

## Acceptance Criteria
- **AC-MEM-1** — `memory::add(text, source)` пишет факт в БД + эмбеддит и кладёт в `memory_vectors`
  (ключ = id). Пустой/whitespace текст отклоняется. Дубликат (точное совпадение текста) не плодится.
- **AC-MEM-2** — `memory::list()` отдаёт все факты (пины сверху, затем по created_at desc).
- **AC-MEM-3** — `memory::set_pinned(id, bool)` / `memory::edit(id, text)` (ре-эмбеддит) / `memory::delete(id)`
  (убирает и из `memory_vectors`).
- **AC-MEM-4** — `memory::context_facts(query, k)` (D2): ВСЕ пины + top-k не-пинов по близости к query;
  обновляет `used_at` подмешанных. Фича-выкл/нет эмбеддера → пусто.
- **AC-MEM-5** — блок памяти инжектится в промпт чата (vault+general) ТОЛЬКО при `aiAgentMemory`=on,
  в injection-маркерах; пустая память → блок не добавляется (MEM-2).
- **AC-MEM-6** — авто-предложение (D1): после обмena быстрая модель извлекает ≤1 факт-кандидат;
  показывается чипом подтверждения; ✓ → `add(source='auto')`, ✗ → отброшен, ничего не пишется (MEM-3).
- **AC-MEM-7** — панель «Память ИИ» (MEM-4): список с пин/правка/удаление/добавление; при размере > `MEM_CAP`
  не-пины старше порога подсвечены «давно не использовался» (D6). Пустое состояние — подсказка.
- **AC-MEM-8** — `aiAgentMemory` персистится (localStorage), дефолт OFF (D5), тоггл в Настройках → AI.
- **AC-MEM-9** — память чистится при смене vault (факты привязаны к vault; индекс пересоздаётся как chat_vectors).

## Тест-стратегия
- Бэкенд (MEM-1): unit на add/list/pin/edit/delete (БД), add→search возвращает факт, дедуп, delete убирает
  из индекса; `context_facts` = пины + top-k (мок-эмбеддер с детерминированными векторами, как chat_log-тесты).
- Инжекция (MEM-2): `build_memory_block` рендерит пины+хиты в маркерах; выкл-флаг → пусто.
- Капчур (MEM-3): авто-предложение возвращает кандидата; confirm→add, reject→no-op (мок fast-LLM).
- UI (MEM-4): рендер списка/пустого; пин/удаление/добавление дёргают команды; тоггл настройки.
- Вероятностную LLM-выдачу (извлечение факта) НЕ гейтим точным текстом — проверяем контракт (≤1 кандидат,
  не пустой при «запомни …»), не содержание.

## Нарезка
- **MEM-1 (бэкенд-фундамент):** миграция 017 + модуль `memory/` (CRUD + эмбеддинг-поиск `context_facts`)
  + команды + pref-стейт + чистка при смене vault. ← этот срез.
- **MEM-2 (инжекция):** `build_memory_block` в RAG-промпт чата за флагом `aiAgentMemory`.
- **MEM-3 (капчур):** явная команда/кнопка «в память» + подтверждённое авто-предложение (fast-LLM).
- **MEM-4 (UI):** панель «Память ИИ» + тоггл Настройки→AI + i18n.

## Не в скоупе v1
Авто-эвикция (D6 — ручная), кросс-vault синк памяти, авто-извлечение без подтверждения (D1),
дедуп по смыслу (только точный текст в v1), TTL-GC (used_at — задел).
