# Vision→AC: «Похожие заметки» + «Прогресс целей» (#35)

> Дешёвые дифференцирующие фичи на готовом фундаменте (RAG-индекс Ф1, `frontmatter_fields`),
> **без** планировщика(#21)/egress(#16)/LLM — работают офлайн. Спроектировано мультиагентным codesign
> (`docs/reviews/ADR_CODESIGN.md`), продуктовые решения зафиксированы владельцем. Без формального
> ADR-номера (бронь ADR-007 — D8). Порядок: **Похожие → Прогресс целей → inline-LLM** (D1).

## Зафиксированные решения

| D | Решение |
|---|---------|
| D1 | Порядок: «Похожие» → «Прогресс целей» → inline-LLM (отдельная спека). |
| D2 | «Похожие» — **дискавери**: показывать всё близкое, ВКЛЮЧАЯ уже связанные (отличие от «Связей», где связанные вычитаются). |
| D3 | Размещение «Похожих» — вкладка в AI-панели рядом со «Связями». |
| D4 | Порог релевантности — **настройка с v1** (слайдер в панели; дефолт 0 = без отсечки; персист `localStorage`). |
| D5 | Маркер цели — **тег `#goal`** (`list_goals` JOIN `tags`+`file_tags` + LEFT JOIN `frontmatter_fields` за `progress`). |
| D6 | Шкала прогресса — **0–100**; `0≤x≤1` → ×100 (0.5→50%); срезать хвостовой `%`. |
| D7 | Тег `#goal` без валидного `progress` — **показать строку без бара + бейдж «нет прогресса»** (не тихий 0%). |
| D8 | Формальный ADR-номер не занимать (спека без ADR). |
| D9 | `/template` — заглушка до отдельной спеки классификатора. |

## «Похожие заметки» (Related Notes) — ✅ реализовано

Второй потребитель max-sim-движка `suggest` (общее ядро `collect_related(exclude_linked)`): для открытого
файла — семантически близкие заметки из сохранённых usearch-векторов **без обращения к embedder-серверу**.
Бэкенд `get_related_notes` (`exclude_linked=false`, без жёсткой отсечки — порог в UI). Фронт — вкладка
«Похожие» (`RelatedView`) со слайдером порога; «вставить связь» дописывает `[[wikilink]]` и **оставляет
строку** (дискавери); клик по заголовку открывает заметку.

- **AC-RN-1** — расчёт без сервера (max-sim из usearch), cap 20, ранжирование по убыванию.
- **AC-RN-2** — включает уже связанные (дискавери, отличие от Suggest).
- **AC-RN-3** — сам файл никогда не в выдаче.
- **AC-RN-4** — нет индекса/векторов → пусто, без краха.
- **AC-RN-5** — относительный порядок сортировки (тестируется на байт-контролируемых фикстурах; семантика — live `#[ignore]`).
- **AC-RN-6** — «вставить» дописывает `[[B]]` (буфер dirty), строку НЕ убирает.
- **AC-RN-7** — i18n RU/EN (заголовок, пустые/loading).
- **AC-X-1** — офлайн-инвариант: работает при выключенных LLM/embedder-серверах.

Тесты: Rust `suggest::related_includes_linked_similar` (+ существующие suggest-тесты на общем ядре);
фронт `stores/related.test.ts` (load/порог/insert-без-удаления), `RelatedView.test.tsx` (рендер + вставка).

## «Прогресс целей» (Goal Progress) — ⏳ следующий срез

Первый кросс-файловый query-консьюмер: `list_goals()` JOIN `tags`+`file_tags` (маркер `#goal`) + LEFT JOIN
`frontmatter_fields` (`key='progress'`); чисто SQL-read, детерминированный (без LLM/сети). Парс шкалы 0–100
(D6) + политика битых значений (D7). Живой пересчёт по `modify` goal-файла — **новый шов реактивности**
(подписка на событие индексатора, не только activePath).

- **AC-GP-1** — сбор целей через JOIN тег+поле; `{path,title,progress:0..100|null}`.
- **AC-GP-2** — парс шкалы 0–100 + валидация (strip `%`, `0.5`→50; невалидное → «без прогресса», не 0%).
- **AC-GP-3** — живой пересчёт по modify goal-файла без планировщика (шов реактивности).
- **AC-GP-4** — детерминизм / зелёный CI без AI и сети.
- **AC-GP-5** — информативное пустое состояние с примером конвенции (`#goal` + `progress`).
- **AC-GP-6** — клик по цели открывает заметку.
- **AC-GP-7** — i18n RU/EN, проценты `tabular-nums`.

## Вне скоупа (явно)

Полные «умные шаблоны» (классификатор) — отдельная спека; их дешёвый срез (ретрив топ-похожих) уже покрыт
«Похожими». inline-LLM — спека `docs/specs/inline-llm.md`, объём M–L, третий по порядку.
