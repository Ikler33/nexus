# Дизайн-бриф: страница «Новости» (News Feed)

> Handoff дизайнеру (дизайн-чат). Источник требований — `docs/specs/news-feed.md` (решения
> владельца D1–D7) + общий `DESIGN_BRIEF.md` (токены/идиомы приложения). Здесь — СТРУКТУРА,
> состояния и компоненты; пиксели/композиция — за дизайнером. Бэкенд и API уже специфицированы —
> данные ниже описаны как реальные поля.

## Назначение страницы

Персональная лента AI-новостей «в одном месте»: раз в сутки приложение собирает доверенные
источники (OpenAI/DeepMind/Mistral/HF/авторские блоги/релизы llama.cpp…), LLM отфильтровывает
нерелевантное и отдаёт **русские** заголовки/резюме + сводку дня. Чтение без перехода по сайтам;
оригинал — по клику.

## Размещение

Отдельная страница приложения (уровень Home/Графа) — пункт в навигации титлбара. Иконка —
газета/лента (lucide `newspaper` или аналог в нашем стиле).

## Структура страницы (сверху вниз)

1. **Шапка-сводка дня** — карточка-дайджест: 5–8 строк RU-текста «главное за сутки» (LLM),
   подпись «Обновлено N мин назад · M статей из K источников» + кнопка **«Обновить»**
   (со спиннером во время прогона). Если часть источников упала — компактная пометка
   «K−E из K источников» с разворачиваемым списком ошибок (no silent caps).
2. **Чипы-фильтры** — темы дня (кластеры от LLM: «Модели», «Инференс/локальный стек»,
   «Исследования», «Релизы»…) + переключатель «Непрочитанные». Темы приходят данными, состав
   не фиксирован.
3. **Лента карточек**, сгруппированных по темам. Карточка:
   - RU-заголовок (кликабелен → оригинал в браузере);
   - 1–2 предложения RU-резюме;
   - мета-строка: источник · время публикации (отн. формат «3 ч назад») · язык оригинала (EN/RU);
   - действия: **прочитано** (toggle, прочитанные визуально гасятся), **«в заметку»**
     (создаёт заметку в vault — короткий toast-фидбек со ссылкой), **скрыть** (опц.);
   - индикатор непрочитанного (точка/акцент слева — как unread-паттерны в приложении).

## Состояния (все обязательны)

| Состояние | Что показываем |
|---|---|
| **Фича выключена** (дефолт!) | Onboarding-CTA: краткое «что это» + кнопка «Включить ленту». Под кнопкой — **консент-строка**: «Запросы будут уходить на N доверенных источников (список)» — спокойный тон, не пугающий (§11: информированное согласие). Включение = тоггл `EgressFeature::NewsFeed`. |
| Включена, первый прогон идёт | Скелетон карточек + статус «Собираю новости…» |
| Пустой день | «Свежих новостей нет» + время следующего автопрогона + кнопка «Обновить» |
| Офлайн-режим (kill-switch egress) | Бейдж «офлайн — лента на паузе», данные последнего прогона остаются читабельны |
| Ошибка прогона целиком | Сообщение + retry; прошлые данные не стираются |
| LLM-часть упала, фиды собраны | Карточки с EN-заголовком + пометка «резюме недоступно» (показываем что есть) |

## Настройки страницы (шестерёнка на странице или секция в Настройках)

- Тоггл фичи (= consent);
- список источников: пресет (вкл/выкл каждый) + «добавить URL» (добавление = consent на хост);
- ключевые слова фильтра (chips-редактор, пресет уже заполнен);
- (инфо) ретенция 30 дней, «навсегда» — через «в заметку».

## Данные (контракт с бэкендом, уже специфицирован)

`NewsItem`: `{ id, source_id, source_title, url, title_ru, summary_ru, topic, published_at,
lang, read: bool }`. Сводка: `{ digest_ru, generated_at, stats: {items, sources_ok, sources_total,
llm_failed} }`. Команды: `get_news(topic?, unread_only?, page)`, `news_mark_read`, `news_to_note`,
`refresh_news`, `get/set_news_config`.

## Тон и идиомы

Тёплый минимализм приложения (токены/типографика как Home/Чат); карточки — плотность списка
заметок, не «журнальный» лейаут; темы-кластеры — рубрики-заголовки в потоке, не табы. Никаких
внешних фавиконок в v1 (приватность/простота) — текстовая метка источника.
