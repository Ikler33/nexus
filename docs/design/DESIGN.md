# Дизайн-документ — Nexus (UI/UX)

> Цель: дать дизайну и фронтенд-коду общий контракт, чтобы вести их **параллельно** с бэкендом.
> Источник истины по логике/IPC: `ARCHITECTURE.md` (v1.1). Этот документ — про визуал, layout, состояния, потоки и a11y.
> Принцип параллельности: дизайн работает против **design-токенов + контракта компонентов + типизированного IPC** (`tauriApi`, §4.1). Пока Rust-бэкенд пишется, UI разрабатывается на моках этих же контрактов.

---

## 0. Как это позволяет параллельную разработку

```
        ОБЩИЕ КОНТРАКТЫ (этот документ + §4.1 ARCHITECTURE)
       ┌───────────────────────────────────────────────┐
       │ design-токены  •  инвентарь компонентов + DoD   │
       │ типы tauriApi (invoke/Channel)  •  состояния    │
       └───────────────────────────────────────────────┘
            ▲                                   ▲
   ДИЗАЙН/ФРОНТ                            БЭКЕНД (Rust)
   React + CSS vars                        Tauri commands
   мок tauriApi (MSW-стиль)                реальные команды
            └──────── стыкуются по контракту ─┘
```

- Фронтенд стартует на **моках** `tauriApi` (те же сигнатуры, что в §4.1) → не ждёт бэкенд.
- Дизайн поставляет **токены** (CSS-переменные) и **состояния** компонентов; код реализует их 1:1.
- Стык — когда обе стороны выполнили свой контракт; интеграция сводится к замене мока на реальные `invoke`/`Channel`.
- **Definition of Done** компонента (см. §10) — единый чек-лист для дизайна и кода.

## 1. Принципы и визуальный язык

| Принцип | Следствие для UI |
|---|---|
| Local-first, спокойный инструмент | Тихая палитра, контент важнее хрома, минимум анимаций |
| Keyboard-first (аудитория power-users) | Любое действие достижимо с клавиатуры; Command Palette — центр |
| Плотность информации | Компактный режим по умолчанию; vault 50k → виртуализация везде |
| AI как слой, не модальность | AI-панель сбоку, не блокирующие модалки; стрим неинвазивен |
| Privacy by default видимо | Явные индикаторы «локально/облако», «индексируется», «синхронизация» |

## 2. Design-токены (CSS Variables — стек §3: CSS Modules + variables)

Токены — **единственный источник истины** для цвета/типографики/отступов. Light/dark — переключением набора переменных на `:root`. Плагины (в т.ч. в iframe) получают те же токены (см. §8).

```css
:root {
  /* Цвет (семантические, не «синий») */
  --color-bg; --color-bg-elevated; --color-surface;
  --color-text; --color-text-muted; --color-border;
  --color-accent; --color-accent-hover;
  --color-success; --color-warning; --color-danger;
  --color-ai;            /* акцент AI-элементов */
  --color-link;          /* [[wikilink]] */
  --color-tag;

  /* Типографика */
  --font-ui; --font-editor; --font-mono;
  --text-xs … --text-2xl;     /* модульная шкала */
  --leading-tight; --leading-normal;

  /* Пространство (4px-сетка) */
  --space-1 … --space-8;
  --radius-sm; --radius-md; --radius-lg;
  --elevation-1; --elevation-2;     /* тени панелей/поповеров */

  /* Движение (сдержанно; уважать prefers-reduced-motion) */
  --motion-fast: 120ms; --motion-base: 200ms; --ease-standard;
}
:root[data-theme="dark"] { /* переопределение цветовых токенов */ }
```

> Контраст текста/фона ≥ WCAG AA (4.5:1). Тема пользователя расширяет токены, не переопределяет компоненты.

## 3. Layout (на базе §4.1 и слотов §7.6)

```
┌────────────────────────────────────────────────────────────────────┐
│ [titlebar / global search]                          [sync] [theme]  │
├──────────┬───────────────────────────────────────┬─────────────────┤
│ Sidebar  │  Editor area (вкладки + сплиты)         │  AI Panel       │
│          │  ┌─[tab][tab][+]──────────────────────┐ │  ┌────────────┐ │
│ FileTree │  │ CodeMirror 6 (source / preview)    │ │  │ Chat       │ │
│ Search   │  │ [editor-bottom: backlinks bar]     │ │  │ Suggestions│ │
│ Tags     │  └────────────────────────────────────┘ │  │ Summary    │ │
│ [slots]  │  (сплит: ещё одна группа вкладок справа)  │  └────────────┘ │
├──────────┴───────────────────────────────────────┴─────────────────┤
│ [status-bar-left: индекс/sync]            [status-bar-right: slots]  │
└────────────────────────────────────────────────────────────────────┘
```

- **Группы/вкладки** (Б12): несколько `EditorGroup`, в каждой свои табы; активная вкладка задаёт контекст AI/backlinks.
- **Слоты плагинов** (§7.6): `left-sidebar`, `right-panel`, `status-bar`, `editor-bottom`, command-palette, context-menu. UI-вью плагина — изолированный iframe; редакторные виджеты — только доверенные in-process (матрица §7.7).
- Панели resizable и сворачиваемые; раскладка персистится в `local`-настройках.

## 4. Инвентарь компонентов (каждый — со всеми состояниями)

Обязательные состояния для каждого: **default / hover / focus / active / disabled / empty / loading / error**. Где есть данные с бэкенда — также **offline** (LLM/сервер недоступен).

| Компонент | Ключевые состояния / заметки |
|---|---|
| **FileTree** (виртуализир.) | empty (пустой vault), 50k (виртуализация flatten-узлов), фильтр/поиск, DnD с auto-expand, контекст-меню, drag-target |
| **Tabs / Split** | активная/неактивная вкладка, dirty-точка, overflow («···»), пустая группа |
| **Editor (CM6)** | source-mode (Ф0), live-preview (поздний эпик), курсор/выделение, `[[wikilink]]`-affordance (подсветка + автокомплит-поповер + клик), `#tag`, поиск-в-документе |
| **Backlinks bar** | список беклинков (с контекстом ±100 симв.), empty «нет связей», loading |
| **Command Palette** | поиск-фильтр, разделы (команды/файлы), горячие клавиши справа, пусто |
| **AI Chat** | пусто (подсказки), streaming (печать токенов + «Стоп»), готово, error/offline, индикатор «локально/☁ облако» |
| **Suggestion cards** (связи) | список с score/причиной, accept/dismiss, «пересчитать», пусто, фоновый «думаю…» |
| **Indexing indicator** | прогресс «N/M чанков», пауза/возобновление, «готово», ошибка эмбеддера (сервер недоступен) |
| **Graph view** | локальный N-hop (по умолчанию), глобальный (с предупреждением о стоимости), loading-layout, фильтры, overlay-слот |
| **Settings** | разделы (AI/редактор/sync/плагины/тема/язык), валидация `*.url`, переключатель cloud-fallback с предупреждением |
| **Plugin Manager** | список/маркетплейс, экран **consent на разрешения** (scoped, с пояснением риска), update/needs-review бейдж, enable/disable без рестарта |
| **Conflict resolver** | three-way diff (диск vs локально vs база), выбор стороны, `.conflict`-уведомление |
| **Onboarding** | выбор/создание vault, проверка LLM-серверов (health), первый прогон индексации |
| **Toasts / Inline errors** | success/warning/danger; «LLM недоступен — проверьте настройки» |

## 5. Ключевые потоки (happy + edge)

1. **Открыть vault** → выбор папки → быстрый файловый индекс (<1s, UI готов) → фоновый парс/эмбеддинг с индикатором. Edge: невалидная папка; чужой vault → плагины `needs-review`.
2. **Поиск** → инкрементально: FTS (мгновенно) + семантика (≤300мс) → результаты с подсветкой. Edge: пусто; эмбеддер offline → деградация до FTS с пометкой.
3. **Чат (RAG)** → вопрос → стрим токенов (throttled) + кнопка **Стоп** → ответ с источниками (кликабельные `<note>`), финализация в историю. Edge: обрыв → ретрай/локальная ошибка; cloud-fallback → видимая плашка «☁ ответ из облака».
4. **Предложения связей** → авто (режим 1, тихо) или по действию (режим 2, LLM) → карточки accept/dismiss. Edge: на save НЕ дёргаем LLM (С-8).
5. **Индексация** → прогресс «N/M», резюмируется после перезапуска. Edge: смена модели → баннер «переиндексация векторов».
6. **Sync/конфликт** → статус в status-bar → при конфликте `.md` открыть resolver; «диск vs грязный буфер» → diff-выбор (С-17).
7. **Установка плагина** → экран разрешений (scoped, риск явно) → consent → enable. Edge: pull принёс код → `needs-review` + показ хэша/diff.

Для каждого потока дизайн поставляет: экраны состояний + тексты (ru/en) + поведение клавиатуры.

## 6. Редактор (отдельный фокус)

- **Ф0 — source-mode**: markdown с подсветкой, `[[wikilink]]` (подсветка/автокомплит/клик), `#tags`, поиск. Backlinks-бар снизу.
- **Live Preview — отдельный эпик** (С-22), НЕ в Ф0: дизайн прорабатывает целевой вид (скрытие разметки вне строки курсора, рендер embeds/таблиц/LaTeX, atomic-ranges-поведение курсора) заранее, но реализация — позже.
- Владение DOM: React монтирует контейнер один раз, CM6 владеет своим поддеревом; смена файла — через `dispatch`, без пересоздания (контракт §4.1/Ф0).

## 7. Стриминг и AI-UX

- Токены приходят по `Channel` (Б10); UI обновляется **throttled** (rAF/≤100мс), не на каждый токен.
- Всегда видна кнопка **Стоп** во время `streaming`; автоскролл «прилипает» к низу только если пользователь внизу.
- Лента чата виртуализирована (длинная история). Источники RAG — кликабельны, ведут к файлу/секции.
- Явная индикация провайдера: значок «локально» по умолчанию, «☁ облако» при fallback (privacy-видимость).

## 8. Тема и темизация плагинов

- Light/dark + кастомные темы — только через переопределение CSS-переменных (§2). FOUC недопустим: тема применяется до первой отрисовки (чтение системной темы + сохранённой).
- Плагинам (вкл. iframe-вью) **прокидываются токены темы** (CSS custom properties передаются в iframe), чтобы их UI совпадал с тёмным/светлым режимом.

---

## 9. i18n и плотность (RU/EN)

- Все строки — ключи i18next; RU-строки в среднем длиннее EN → layout тянется, без обрезки/overflow (тест на RU).
- Плюралы one/few/many; числа/даты — `Intl`; сортировка — `Collator` (§9 ARCHITECTURE).
- Плагинные строки — namespace `plugin:<id>:<key>`.
- Смена языка — без перезапуска, без потери состояния.

## 9a. Доступность (a11y) — с Ф0, не «потом»

- Полная клавиатурная навигация; Command Palette как универсальный вход; видимый focus-ring (токен).
- ARIA: дерево (`treeitem`/`aria-expanded`), вкладки (`tablist`), стрим (`aria-live=polite`).
- Контраст ≥ AA; уважать `prefers-reduced-motion`; цель тача/клика ≥ 24px.

## 10. Контракт «дизайн ↔ код» и Definition of Done компонента

- **Токены** — source of truth; код не хардкодит цвета/отступы.
- **Storybook** (или аналог): каждый компонент — со всеми состояниями из §4; это и есть точка стыка дизайна и кода.
- **Контракт данных** = типы из §4.1 (`tauriApi`, `StreamEvent`, `EditorGroup`, `ChatSession`…). Фронт мокирует их, дизайн опирается на те же формы.

**DoD компонента (общий для дизайна и кода):**
1. Реализованы состояния default/hover/focus/active/disabled/empty/loading/error (+offline где есть данные).
2. Только токены (нет хардкода); light/dark проверены.
3. Клавиатура + ARIA + контраст (§9a).
4. i18n: ключи в ru и en; проверено на длинных RU-строках.
5. Виртуализация, если список может быть большим (дерево/чат/результаты).
6. Привязка к контракту `tauriApi` (или его моку) — без прямых `invoke` вне `lib/tauri-api.ts`.
7. Storybook-стори покрывает состояния; визуальный снапшот стабилен.
