## 0. Общие правила для всех экранов

Эти правила обязательны и не повторяются в каждом разделе ниже.

### 0.1. Источник истины визуала — токены Hermes
**Запрещено вводить новые цвета.** Любой цвет/отступ/радиус/шрифт/моушн — только через существующий `var(--…)` из `tokens.css`. Список доступных токенов:

**Поверхности / нейтрали (меняются по теме автоматически):**
`--color-bg` (холст редактора/страницы), `--color-chrome` (титлбар/сайдбар/статус-бар/таб-стрип), `--color-chrome-2` (акценты chrome/разделители), `--color-bg-elevated` (карточки, попап-меню, поповеры), `--color-surface` (инпуты, утопленные чипы), `--color-surface-hover` (ховер), `--color-selected` + `--color-selected-hover` (ЕДИНЫЙ цвет выделения — строки дерева, активные кнопки рейла/титлбара).

**Текст:** `--color-text` (основной), `--color-text-muted` (вторичный), `--color-text-faint` (третичный / иконки в покое).

**Границы:** `--color-border` (волоски), `--color-border-strong` (заметные границы/ховер инпута).

**Акцент (терракота по умолчанию; свапается `data-accent`):** `--color-accent`, `--color-accent-hover`, `--color-accent-soft` (мягкая подложка/focus-ореол), `--color-accent-rim` (кольцо focus-within), `--color-on-accent` (текст на акценте).

**AI/info-слой (teal):** `--color-ai`, `--color-ai-soft`. **Семантика:** teal = «это сделано/обрабатывается LLM». Любой LLM-генерируемый контент маркируется teal-обработкой (см. 0.4).

**Ссылки/теги:** `--color-link` + `--color-link-soft` (`[[wikilinks]]`), `--color-tag` + `--color-tag-soft` (`#tags`, sage).

**Статусы:** `--color-success` (sage), `--color-warning` (amber), `--color-danger` + `--color-danger-soft` (red-clay).

**Тени/elevation:** `--elevation-1` (мягкая, карточки), `--elevation-2` (поповеры/модалки), `--chrome-shadow`, `--tab-shadow`.

**Фокус/выделение:** `--focus-ring` (готовая двухслойная обводка для `:focus-visible`), `--selection` (текстовое выделение).

**Типографика:** семейства `--font-ui` (Onest — весь UI), `--font-mono` (JetBrains Mono — мета/числа/код/время), `--font-serif` (Source Serif 4 — проза: приветствие, AI-проза, открытые вопросы, крупные числа-статы). Шкала: `--text-xs 11 · --text-sm 12 · --text-base 13 · --text-md 14 · --text-lg 16 · --text-xl 20 · --text-2xl 26`. Высоты строк `--leading-tight 1.3`, `--leading-normal 1.55`.

**Сетка/геометрия:** `--space-1…8` (4·8·12·16·20·24·32·48), `--radius-sm` (7) / `--radius-md` (10) / `--radius-lg` (14), пилюли `99px`. Высота строки списка `--row-h` (28; компакт 24) под управлением `--density`.

**Моушн:** `--motion-fast` (140ms — ховеры, цвет), `--motion-base` (220ms — появление, трансформ), ease `--ease-standard`. Тактильное нажатие кнопок `transform: scale(0.94–0.97)`. Карточки приподнимаются на ховере. Списки появляются stagger-каскадом. **Всё декоративное движение под `@media (prefers-reduced-motion: reduce)` отключается.**

### 0.2. Темы
Переключение темы — атрибут `data-theme` на `<html>`: `light` («old paper», тёплый крем, `--ui-hue 85`) и `dark` («warm clay», тёплый уголь, `--ui-hue 50`). Дефолт без атрибута = light. Поскольку компоненты ходят только через токены, экран должен корректно выглядеть в обеих темах **без отдельной вёрстки** — в разделах ниже «Поведение в светлой/тёмной теме» описывает только смысловые нюансы (например: в dark акцент светлее на +0.10 L, danger/warning ярче; teal-бордюры читаемы за счёт `color-mix`). Смена темы — кросс-фейд 320ms (класс `.theme-anim` на `<html>`, gated `prefers-reduced-motion`).

### 0.3. Акцент и плотность
`data-accent` ∈ `amber` (деф.) / `teal` / `sage` / `clay` перетинтовывает акцентные элементы. **Не заливать строки/карточки насыщенным акцентом** — акцент держим для иконок, индикаторных полос (3px слева / 2px сверху), focus-колец и primary-кнопок; фон активного состояния — нейтральный `--color-selected`. Плотность — через `--row-h` (`--density`): экраны со списками должны переживать переключение compact/comfortable без поломки.

### 0.4. Маркировка LLM-контента (сквозной паттерн)
Любой блок, чьё содержимое сгенерировано моделью, визуально отличается от статики (паттерн `.h-card.ai` из `home.css`):
- бордюр карточки = `color-mix(in oklch, var(--color-ai) 30%, var(--color-border))`;
- 2px teal-полоса слева (`::before`, `background: var(--color-ai)`, `opacity .5`);
- бейдж `AI` рядом с заголовком: `--text` ~9.5px, вес 700, цвет `--color-ai`, фон `--color-ai-soft`, бордюр `color-mix(… --color-ai 40% …)`, радиус 99px;
- иконка заголовка в `--color-ai` (вместо `--color-text-muted` у статики).

### 0.5. Состояния (обязательны для каждого виджета/экрана)
Каждый компонент проектируется в состояниях: **пустое** (нет данных — приветливый es-блок, не голый ноль), **загрузка** (для LLM — анимированный бренд-марк/«дыхание», НЕ спиннер; для статики — мгновенно, скелет не нужен), **ошибка** (баннер/строка с `--color-danger`, действие «повторить»), **успех/данные**, и для LLM-виджетов — **из кэша** (метка «обновлено N назад» + «обновить») и **обновляется** (мягкий teal-пульс поверх старого контента, без блокировки страницы).

### 0.6. A11y (база для всех экранов)
- **Фокус:** только `:focus-visible`, обводка = `--focus-ring` (двухслойная: 2px фон + 2px ободок), радиус `--radius-sm`. Видим на обеих темах.
- **Клавиатура:** всё кликабельное достижимо Tab/Shift+Tab; списки — стрелки ↑/↓ + Enter; модалки — Esc закрывает, фокус-трап внутри, возврат фокуса на триггер; роли ARIA (`role="dialog"`, `aria-modal`, `aria-label`).
- **Контраст:** токены рассчитаны на WCAG AA; нельзя ставить `--color-text-faint` на `--color-surface` для важного текста — faint только для декоративного/мета. Статусы дублировать не только цветом (иконка/текст), т.к. цветовая индикация stale/конфликтов критична.
- **Моушн:** уважать `prefers-reduced-motion`. **Egress/приватность:** любое сетевое действие имеет явную видимую метку (см. чат и News Feed).

### 0.7. Оболочка (chrome) — общая для in-app экранов
Home, настройки, News Feed, конфликты, карта компетенций живут внутри той же оболочки, что и воркспейс: CSS-grid `38px титлбар / 1fr тело / 26px статус-бар`. Титлбар (`.titlebar`, Liquid-Glass `backdrop-filter: blur(16px)` поверх `--color-chrome`): traffic-lights, тоггл сайдбара, бренд-марк (логотип-созвездие) + «Nexus», центральная пилюля **глобального поиска** (`⌘K`), правая группа инструментов: граф (`⌘⇧G`), текст-тоггл `RU / EN` (активная сторона — `--color-accent`), тема (sun/moon, кросс-фейд), AI-панель. Активная кнопка инструмента = `--color-selected` фон + акцентная иконка.

### 1.3. Классы виджетов (определяют состояния и моушн)
- **Static** (Недавние файлы, Статистика) — данные прямо из vault, без обработки. Мгновенно; реагируют на `vault.on(modify)` / `on-open`. Нет состояния «загрузка»/«обновляется».
- **Dynamic** (Прогресс целей, Stale radar слой 1) — вычисление из frontmatter/метаданных (scoring, агрегация), без LLM. Мгновенно; пересчёт при `modify`.
- **LLM-powered** (Daily brief, Открытые вопросы, Context drift, Stale radar слой 2) — генерируется локальной моделью, **кэшируется**, обновляется по триггеру. Несут teal-маркировку (0.4) и состояния кэша/обновления.

Режимы обновления LLM-виджетов (отражать в UI через card-act и сабтекст): **on-open** (Daily brief — при открытии, если кэш устарел), **on-vault-change** (с дебаунсом ~30с), **scheduled** (Context drift — раз/сутки в фоне), **manual** (Открытые вопросы, Stale-слой-2 — только по кнопке). Архитектура кэша: каждый LLM-виджет хранит результат + timestamp; при открытии сверяется mtime исходных файлов; не изменились → кэш без запроса; изменились → перегенерируются только затронутые виджеты.

### 1.5. Ключевые состояния
- **Пустое (новый/пустой vault):** зоны static показывают приветливые es-заглушки («Пока нет заметок — создайте первую», CTA на quick-action). LLM-виджеты — нейтральная подсказка + кнопка запуска, не пугающий ноль.
- **Загрузка LLM-виджета:** контент области виджета заменяется анимированным бренд-марком (созвездие, «дыхание»), НЕ спиннером; остальной дашборд интерактивен.
- **Из кэша:** card-act показывает «обновлено N назад» (mono, faint); контент отрисован сразу.
- **Обновляется:** старый контент остаётся видим, поверх — мягкий teal-пульс/прогресс-нить (`--color-ai`), без layout-сдвига и блокировки.
- **Ошибка LLM (модель недоступна):** в теле AI-карточки — компактный inline-баннер по паттерну `.ai-banner.danger` (`--color-danger-soft` фон, бордюр `--color-danger`): «Локальная модель недоступна» + ссылка «повторить». Статические виджеты при этом работают как ни в чём не бывало.
- **Успех:** данные + актуальный timestamp.

### 1.6. Взаимодействия
Hero-search: ввод `/` → режим vault-поиска, `?` → режим вопроса к LLM (открывается палитра/раскрывается результат). Quick-actions запускают команды (создать заметку и т.д.); «Переиндексировать» = перегенерировать все LLM-виджеты (массовый refresh с per-виджет индикаторами). Клик по строке (recent/stale/open-question) открывает заметку в активной панели воркспейса. card-act «обновить» → ручная регенерация конкретного виджета. card-act «все →» → переход в полный список/раздел. Все списки навигируемы стрелками.

### 1.7. Поведение в светлой/тёмной теме
Всё на токенах → перекрашивается автоматически. Нюансы: свечения dot/stale-dot строятся через `box-shadow` от статус-токенов и в dark выглядят ярче (токены danger/warning/success в dark ярче) — это ожидаемо и желаемо (радар «горит»). Teal-бордюры AI-карточек используют `color-mix` с `--color-border`, поэтому читаемы в обеих темах. Greeting-`em` и focus-кольца берут текущий `--color-accent`, т.е. реагируют и на `data-accent`.

### 1.8. Токены Hermes (точечно)
Поверхности виджетов `--color-bg-elevated`; утопленные (stat/track) `--color-surface`; ховер строк `--color-surface-hover`. Бордюры `--color-border` / `--color-border-strong`. AI-обработка — `--color-ai` / `--color-ai-soft` (+ `color-mix`). Статусы радара/прогресса — `--color-danger` / `--color-warning` / `--color-success` / `--color-accent` / `--color-ai`. Текст: заголовки/числа `--color-text`, проза/мета `--color-text-muted`, время/хинты `--color-text-faint`. Шрифты: проза и крупные числа `--font-serif`, мета/время/проценты `--font-mono`, остальное `--font-ui`. Радиусы `--radius-lg` (карточки) / `--radius-md` (qa, stat) / `--radius-sm` (строки) / `99px` (чипы, dot-трек). Отступы из `--space-*`, моушн `--motion-fast/base` + `--ease-standard`, тени `--elevation-1`. Focus `--focus-ring`.

### 1.9. A11y
Дашборд — лендинг, поэтому первый Tab-стоп = hero-search. Все карточки — landmark-регионы (`<section aria-labelledby>` от sec-label). Списки = `role="list"`/`listitem`, навигация стрелками, Enter открывает. Статусные точки stale/live дублируются текстом (дни / «онлайн») — не только цвет. card-act-кнопки — настоящие `<button>` с `aria-label` («Обновить Daily brief»). Анимация «дыхания» при загрузке — под reduced-motion заменяется статичной надписью «Генерируется…». Контраст: serif-проза идёт на `--color-text-muted` (AA на elevated), но ключевые слова поднимаются в `--color-text`.

## 2. Полный раздел настроек  ⭐ приоритет 2

> Расширяет существующую tweaks-панель (тема/акцент/плотность уже есть) до полноценных настроек. Закрывает пункт BACKLOG «Полный раздел настроек (общие · горячие клавиши (ремап) · LLM-эндпоинты в UI вместо `.nexus/local.json`)».

### 2.1. Цель

### 2.2. Layout / зоны

Левая нав (`--color-chrome`, строки как `.tree-row`: активная = `--color-selected` + 3px акцент-полоса слева + акцентная иконка + вес 500). Правая — `--color-bg-elevated`/`--color-bg`, скролл, секции через `.sec-label`. Ширина диалога ~760–900px, `--radius-lg`.

### 2.3. Компоненты (паттерны полей)
Единый «settings-row»: слева лейбл (`--text-md`, `--color-text`) + опц. описание (`--text-sm`, `--color-text-muted`); справа контрол. Контролы:

- **Свотчи акцента** (amber/teal/sage/clay): 4 кружка, заливка = соответствующий вычисленный `--color-accent` пресета, активный обведён `--color-accent-rim`.

- **Текст-инпут** (`.field`): `--color-surface`/`--color-bg-elevated`, бордюр `--color-border`, `--radius-md`, focus-within → `--color-accent-rim` + `0 0 0 3px var(--color-accent-soft)`. Для путей — с кнопкой «Обзор…».
- **Select/dropdown** (язык RU/EN, модель из списка): поповер `--color-bg-elevated` + `--elevation-2`.
- **Keycap-редактор** (ремап): см. 2.4.

### 2.4. Раздел «Горячие клавиши» (ремап)
keymap уже data-driven → ремап трекабелен. Таблица команд: строки `command → текущая комбинация → [изменить]`. Комбинация отрисована компонентом `.kbd` (mono, фон `--color-bg`, бордюр `--color-border`, радиус 4px), модификаторы отдельными капсами (`⌘` `⇧` `\`). 
- **Захват:** клик «изменить» → строка входит в режим записи (бордюр → `--color-accent-rim`, текст «Нажмите сочетание…»), следующий keydown пишется. Esc отменяет.
- **Конфликт:** если комбинация занята — inline-предупреждение `--color-warning` («Уже назначено: Граф») + выбор «переназначить/отмена».
- **Сброс:** «вернуть по умолчанию» (faint-кнопка) на строке и на разделе.
Базовый набор для отображения: `⌘K` палитра, `⌘⇧G` граф, `⌘\` сплит, `⌘R` чтение, `⌘E` edit/preview, `⌘S` сохранить, `Esc` выход/закрытие.

### 2.5. Раздел «LLM-эндпоинты»
Выносит `.nexus/local.json` в UI. Группы полей: **Chat-модель** (URL эндпоинта, имя модели, опц. ключ), **Embedding-сервер(ы)** (URL, размерность — read-only из ответа), **Реранкер** (опц.). Для каждого эндпоинта:
- text-инпут URL + статус-индикатор **проверки соединения**: точка + подпись по паттерну provider-пилюли — `--color-success` «доступен» / `--color-warning` «проверяется» (маленький спиннер допустим, как в онбординге) / `--color-danger` «недоступен»;
- кнопка «Проверить» (secondary);
- ключ — поле type=password с тогглом видимости; примечание «хранится в системном keychain, не в файле проекта» (`--text-xs`, faint).
**Egress-секция:** тоггл «Разрешить облако/web (egress)» с явным предупреждающим описанием (`--color-warning` рамка-баннер `.ai-banner.warn`): «данные покинут устройство только при включении». По умолчанию OFF (local-first, ADR-005).

### 2.6. Ключевые состояния
- **Загрузка:** значения подтягиваются из конфига; до готовности — лёгкий скелет полей.
- **Проверка эндпоинта:** «проверяется» (warning + спиннер) → «доступен»/«недоступен».
- **Ошибка сохранения/валидации:** inline под полем (`--color-danger`, `--text-sm`), поле получает danger-бордюр.
- **Успех:** тихое подтверждение (изменения применяются сразу для оформления; для эндпоинтов — toast «сохранено»). Кнопка «Сброс к умолчанию» в каждом разделе.
- **Пустое:** для нового пользователя поля LLM пусты с плейсхолдерами-примерами (`http://192.168.0.172:8080`).

### 2.7. Взаимодействия
Левая нав переключает раздел (стрелки + Enter). Оформление применяется мгновенно (живой превью апп под модалом — как teal перетинтовывает интерфейс). Ремап — запись клавиш. «Проверить» — асинхронная проба с индикатором. Esc / ✕ закрывает; несохранённые правки эндпоинтов — подтверждение «применить/отменить».

### 2.8. Поведение в светлой/тёмной теме
Полностью на токенах. Сегменты/свотчи/тогглы используют `--color-selected`/`--color-accent`, корректны в обеих темах. Статус-точки эндпоинтов — статус-токены (ярче в dark). Модал-scrim — `oklch(0 0 0 / .32)`, поверх — `--color-bg-elevated` + `--elevation-2`.

### 2.9. Токены
Нав `--color-chrome` + `--color-selected`/3px `--color-accent`; панель `--color-bg-elevated`; поля `--color-surface` + focus `--color-accent-rim`/`--color-accent-soft`; keycaps `--font-mono` + `--color-bg`/`--color-border`; статусы `--color-success/warning/danger`; egress-баннер `.ai-banner.warn`-паттерн (`--color-warning`); тексты `--color-text`/`--color-text-muted`/`--color-text-faint`; радиусы `--radius-lg/md`; `--elevation-2`; `--focus-ring`.

### 2.10. A11y
Модал: `role="dialog" aria-modal`, фокус-трап, Esc-закрытие, возврат фокуса. Нав — `role="tablist"`, разделы — `role="tabpanel"`. Тогглы — нативный `aria-checked`/`role="switch"`. Поля — связанные `<label for>`. Ремап доступен с клавиатуры (Enter входит в запись, Esc отменяет). Статус эндпоинта дублируется текстом, не только цвет точки. Поле ключа — `aria-describedby` про keychain.

## 3. Чат → агент: режимы vault / общий / web  ⭐ приоритет 3 (vision-critical)

> Расширяет существующую AI-панель. Сейчас чат всегда грунтуется в vault; задумка — ассистент с выбором режима и tool-use (web-search через self-hosted SearXNG). Egress opt-in. Источник: BACKLOG «Чат → агент».

### 3.1. Цель
Дать пользователю явный контроль над тем, **откуда** ассистент берёт знание: только локальная база (vault-RAG), общая модель без vault, или агент с web-поиском. Главное — **прозрачность egress**: пользователь всегда видит, покидают ли данные устройство.

### 3.2. Layout / зоны
В рамках существующей AI-панели (`.ai-panel` на `--color-chrome`, раскладки side/bottom/overlay). Структура: header (`.ai-head`: sparkles + «AI-ассистент» + провайдер-пилюля) → **переключатель режимов** (новое) → лента сообщений (`.ai-body`) → композер (`.ai-composer`).

```
┌ AI-ассистент            [provider: ◉ локально] ┐
│ [ ◉ Vault ] [ Общий ] [ Web ]   ← сегмент режима │
│ ───────────────────────────────────────────────│
│  ... лента сообщений (msg.user / msg.ai) ...    │
│  (для web-ответа: tool-step «🔎 поиск…» +       │
│   цитаты-источники src-cards)                   │
│ ───────────────────────────────────────────────│
│  [ композер ]                            [→]    │
│  ⏎ отправить · ⎋ стоп        ☁ egress: SearXNG  │← egress-индикатор
└─────────────────────────────────────────────────┘
```

### 3.3. Компоненты
- **Сегмент режима** (новый, под header): три пилюли `Vault / Общий / Web` (паттерн сегмента `.tb-btn`). Активная = `--color-selected` + акцентная иконка/текст. Иконки: vault = база/папка, общий = чат, web = глобус. Под активным — короткая подпись режима (`--text-xs`, faint): «грунтуется в вашей базе» / «общие знания модели» / «ищет в интернете и цитирует».
- **Провайдер-пилюля** (`.provider`, существующая): `.local` — иконка `--color-success`; `.cloud` — `--color-warning` фон/бордюр (когда модель облачная). Отражает где крутится модель.
- **Egress-индикатор** (ключевой новый элемент): в `composer-foot`. Для Vault/Общий локально — нейтрально/скрыт. Для **Web** или облачной модели — заметная метка: иконка ☁/глобус + текст «egress: SearXNG (VPS)» в `--color-warning`, с тултипом «запросы уходят на ваш self-hosted поиск». При выключенном egress (см. настройки) режим Web заблокирован с подсказкой «Включите egress в настройках».
- **Лента сообщений** (существующая): `.msg.user` (пузырь `--color-accent-soft`, выровнен вправо) / `.msg.ai` (прозрачный, слева). Стриминг — плавный char-by-char (RAF ~62 cps, мигающий каре `.cursor-blink` в `--color-ai`), кнопка Stop.
- **Tool-step (web-режим, новый):** перед ответом — компактная строка-«шаг агента»: иконка лупы + «Ищу: <query>» → «Найдено N источников», teal-акцент (это действие LLM). Сворачиваемо.
- **Источники-цитаты:** существующие 3 стиля (`.src-cards` карточки / `.src-chips` чипы / `.src-ref` сноски). Для vault — ссылки на заметки (`--color-link`); для web — внешние ссылки с фавиконкой-плейсхолдером и доменом. Номер источника `.sc-num` на `--color-ai`.
- **Композер** (`.composer-box`): один скруглённый бокс `--color-bg-elevated`, focus-within → `--color-accent-rim` + accent-soft кольцо. Send-кнопка `--color-accent`; во время стрима → `.stop` (`--color-danger`).
- **Баннеры:** `.ai-banner.danger` (модель офлайн) / `.ai-banner.warn` (web-режим включён — «ответы могут содержать внешние данные»).

### 3.4. Ключевые состояния
- **Пустое (Vault):** `.chat-empty` — анимированный бренд-марк в `--color-ai-soft` + заголовок + suggestion-пилюли «спросить про…». 
- **Пустое (Web, egress off):** баннер-подсказка «Web-режим требует egress» + кнопка «Открыть настройки».
- **Загрузка/думает:** анимированный «thinking» бренд-марк (~1.3с), затем стрим.
- **Web-поиск:** tool-step «ищу…» с teal-пульсом до прихода источников.
- **Ошибка (модель/поиск недоступны):** `.ai-banner.danger` + «повторить».
- **Стоп:** Stop отменяет RAF, сообщение помечается прерванным.

### 3.5. Взаимодействия
Смена режима пилюлей мгновенно меняет контекст следующего сообщения (и подпись/egress-индикатор). При выборе Web с выключенным egress — режим визуально disabled + подсказка. Клик по источнику: vault → открыть заметку в воркспейсе; web → открыть во внешнем браузере (с явной иконкой внешней ссылки). Stop прерывает стрим/поиск. Раскладка панели (side/bottom/overlay) переключается в tweaks.

### 3.6. Поведение в светлой/тёмной теме
AI-слой `--color-ai` ярче в dark (teal читаем). Egress-warning `--color-warning` ярче в dark — это усиливает заметность egress, что желательно. Провайдер-пилюля `.cloud` использует amber-подложку, корректную в обеих темах.

### 3.7. Токены
AI-слой `--color-ai`/`--color-ai-soft` (бейджи, tool-step, источники, каре). Egress/web-предупреждение `--color-warning` + `.ai-banner.warn`. Провайдер `.local`→`--color-success`, `.cloud`→`--color-warning`. Активный режим `--color-selected` + `--color-accent`. Пузырь юзера `--color-accent-soft`. Композер focus `--color-accent-rim`/`--color-accent-soft`; send `--color-accent`/stop `--color-danger`. Источники-ссылки `--color-link`. Текст muted/faint по иерархии. `--font-mono` для номеров источников/kbd.

### 3.8. A11y
Сегмент режима — `role="radiogroup"`/`radio`, стрелки переключают, активный `aria-checked`. **Egress-индикатор — не только цвет:** обязательны иконка + текст «egress включён/выключен» + `aria-live` объявление при первом уходе данных в сеть («Запрос отправлен на внешний поиск»). Заблокированный Web-режим — `aria-disabled` + объяснение. Стрим-сообщение — `aria-live="polite"` (без спама на каждый символ — батчить). Stop достижим клавиатурой. Источники — список ссылок с распознаваемой ролью (внутр./внешн.).

## 4. Inline LLM в редакторе (ghost-text)  ⭐ приоритет 4

> CM6-расширение со стримом в документ. Ядро AI-PKM. Источник: BACKLOG «Inline LLM», концепт §«Умные шаблоны» (ghost-text/панель).

### 4.1. Цель
Предложения модели прямо в потоке письма, не прерывая его: продолжение/структура/генерация по выделению. Ghost-text появляется серым, `Tab` принимает, `Esc` отклоняет — можно просто продолжать печатать и не заметить.

### 4.2. Layout / зоны
Внутри редактора (CM6, source-mode, центр-measure `--editor-measure` 760px). Три точки появления (соответствуют моментам срабатывания умных шаблонов):
- **Ghost-text inline** — серый текст-призрак ниже/после курсора, в потоке.
- **Inline-тулбар по выделению** — маленькая плавающая панель над выделенным фрагментом (генерация/переписать/продолжить).
- **Панель выбора структуры** — для сложных типов заметки (статья/проект/ретро): 2–3 варианта скелета с превью (всплывающая карточка `--color-bg-elevated` + `--elevation-2`).
- **Slash-команда** `/` в пустой строке / `/template` — открывает компактное меню inline-команд (паттерн палитры в миниатюре).

### 4.3. Компоненты
- **Ghost-text:** цвет `--color-text-faint`, тот же `--editor-font`/размер, без фона; курсор остаётся в `--color-accent` (caret-color). После текста — крошечный хинт `Tab` (`.kbd`-стиль) и `Esc`.
- **Accept/Reject хинт-бар:** под ghost-text строка `--text-xs` faint: «`Tab` принять · `Esc` отклонить».
- **Inline-тулбар (по выделению):** пилюля-группа `--color-bg-elevated` + `--elevation-1`, кнопки прозрачные (ховер `--color-surface-hover`), иконки в `--color-accent` для primary-действия; teal-акцент сигнализирует, что результат — от LLM.
- **Панель структуры:** карточки-варианты `.sg-card`-паттерн (`--color-surface`, бордюр), заголовок варианта `--color-link`/`--color-text`, превью секций faint; кнопки выбора `.sg-btn`/`.sg-btn.primary` (`--color-accent`).
- **Slash-меню:** список команд (как палитра), активная строка `--color-accent-soft`, kbd-хинты.

### 4.4. Ключевые состояния
- **Idle:** ничего; редактор обычный.
- **Генерация:** ghost-text стримится посимвольно (тот же RAF-плавный приём, цвет faint); крошечный teal-пульс у строки «генерируется».
- **Предложение готово:** ghost-text + accept/reject хинт.
- **Принято:** ghost-text фиксируется обычным `--color-text`, хинт исчезает, документ помечается dirty (•).
- **Отклонено (Esc):** ghost-text исчезает мгновенно, курсор на месте.
- **Ошибка (модель недоступна):** тихая нотификация-строка `--color-danger` у курсора «модель недоступна», без модала.
- **Пустое (нет контекста):** для `/template` без распознанного типа — fallback на панель выбора.

### 4.5. Взаимодействия
Триггеры: пауза в печати ~2с (предложение продолжения), `/`-команда, выделение → тулбар, `/template`. `Tab` принять / `Esc` отклонить — не должны конфликтовать с обычным Tab (ghost активен → Tab принимает; нет ghost → обычная табуляция). Принятые/отклонённые предложения логируются для обучения (невидимо для UI). Стрим можно прервать `Esc`.

### 4.6. Поведение в светлой/тёмной теме
Ghost-text = `--color-text-faint`, в обеих темах читаемо-приглушён (faint рассчитан как третичный, но различим). caret в `--color-accent`. teal-пульс генерации ярче в dark.

### 4.7. Токены
Ghost-text `--color-text-faint`; caret/`primary` `--color-accent`; LLM-маркеры `--color-ai`; хинты `.kbd` (`--font-mono`, `--color-bg`/`--color-border`); inline-панели `--color-bg-elevated` + `--elevation-1/2`; карточки структуры `.sg-card`/`.sg-btn` (`--color-surface`/`--color-accent`); ошибка `--color-danger`. Шрифт ghost — `--editor-font`/`--editor-size`.

### 4.8. A11y
Ghost-text — `aria-live="polite"` регион с объявлением предложения (один раз, не по символам). Хинт «Tab принять/Esc отклонить» виден текстом, не только подразумевается. Принятие/отклонение доступно только с клавиатуры — это и есть основной путь. Контраст faint-ghost: т.к. это временное предложение, допустимо faint, но при `prefers-reduced-motion`/высоком контрасте — поднимать до `--color-text-muted`. Inline-тулбар достижим (например `⌘.`), не только мышью.

## 5. News Feed (Лента новостей)  ⭐ приоритет 5

> Отдельная страница-агрегатор: RSS/открытые API → 2-этапный фильтр keyword→LLM → связь с vault. Scheduled раз/сутки. Ресурсы opt-in. Источник: BACKLOG «News Feed», концепт §«Лента новостей».

### 5.1. Цель
Персональная лента по интересам пользователя без алгоритмов платформ: приложение тянет источники, LLM фильтрует по профилю и оставляет релевантное, связывает статьи с заметками vault. Обновляется раз/сутки в фоне; пользователь открывает — лента готова.

### 5.2. Layout / зоны
Полноэкранная страница в оболочке 0.7 (нав-пункт в сайдбаре «Лента», рядом с Home). Центрированная колонка (`max-width ~1080px`, padding `--space-7`), как Home:

```
┌ Лента новостей            [обновлено сегодня 09:14] [↻ обновить сейчас] ┐
│  Фильтры:  [ Все ] [ Модели ] [ Инструменты ] [ Статьи ] [ Связано с базой ] │← кластеры по содержанию
│ ── сегодня ─────────────────────────────────────────────────────────  │
│  ┌ карточка статьи ──────────────────────────────────────────────┐    │
│  │ Заголовок              [источник-чип] [новизна: новое/покрыто]  │    │
│  │ Одно предложение резюме (serif)                                  │    │
│  │ 🔗 касается твоей заметки «KV cache» →                           │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│  ...                                                                     │
└──────────────────────────────────────────────────────────────────────────┘
```

### 5.3. Компоненты
- **Хедер-строка:** заголовок + статус обновления (`--font-mono`, faint, «обновлено сегодня 09:14») + кнопка «↻ обновить сейчас» (secondary, ручной запуск вне расписания).
- **Фильтр-чипы (кластеры по содержанию, не по источнику):** ряд пилюль `.h-chip`-паттерн; активный = `--color-selected` + акцентный текст. Спец-чип «Связано с базой».
- **Карточка статьи** (`.h-card`): заголовок (`--text-md`, вес 600, кликабелен → внешняя ссылка) + **источник-чип** (мono-пилюля, имя источника: «HF Papers», «arxiv cs.CL», «Simon Willison») + **бейдж новизны**: «новое» (`--color-success`-чип) / «уже покрыто в базе» (`--color-text-faint`-чип). Тело — **одно предложение резюме** от LLM в `font-serif` 15px muted (это LLM-контент → опц. teal-акцент карточки `.h-card.ai`). Низ карточки — **связь с vault**: строка «🔗 касается твоей заметки «…» →» (иконка `--color-link`, ссылка на заметку) или «создать заметку» (если связи нет).
- **Egress-плашка (важно):** т.к. лента тянет внешние источники — заметная метка вверху страницы (как egress-индикатор чата): «Источники получаются из сети» (`--color-warning`); при выключенных ресурсах в настройках — лента пуста с CTA «Включить источники».
- **Двухэтапность (отражать в пустых/прогресс-состояниях):** этап 1 keyword-фильтр (мгновенно, без LLM), этап 2 — LLM-оценка остатка (прогресс «оценено N из M»).

### 5.4. Ключевые состояния
- **Пустое (первый запуск / нет источников):** es-блок «Добавьте источники и ключевые слова» + CTA в настройки ленты.
- **Загрузка (фоновый прогон):** мягкий прогресс-бар вверху «обновляю ленту — оценено 24/60» (teal); старая лента остаётся видимой.
- **Готово:** карточки, сгруппированные по дню/кластеру; stagger-появление.
- **Пусто после фильтра:** «За сегодня ничего релевантного» (faint).
- **Ошибка источника:** per-источник тихая отметка (не рушит всю ленту), общий баннер `.ai-banner.warn` если сеть недоступна.

### 5.5. Взаимодействия
Клик по заголовку → внешний браузер (явная внешняя ссылка). Клик по «касается заметки» → открыть заметку; «создать заметку» → новая заметка с контекстом. Фильтр-чипы сужают ленту. «Обновить сейчас» — ручной прогон с прогрессом. Источники/ключевые слова/расписание настраиваются в настройках (раздел «Лента»).

### 5.6. Поведение в светлой/тёмной теме
На токенах. Источник-чипы (`--color-tag`/mono) и бейджи новизны (`--color-success`/faint) корректны в обеих. Egress-плашка `--color-warning` заметнее в dark.

### 5.7. Токены
Карточки `.h-card` (+ опц. `.h-card.ai` teal для LLM-резюме); чипы-источники `--color-tag`/`--color-tag-soft` + `--font-mono`; новизна `--color-success` / `--color-text-faint`; связь с vault `--color-link`; статус обновления `--font-mono` faint; egress `--color-warning` + `.ai-banner.warn`; резюме `--font-serif` muted; фильтры `--color-selected`/`--color-accent`.

### 5.8. A11y
Лента — `role="feed"`, карточки `article` с `aria-labelledby` (заголовок). Внешние ссылки — распознаваемая роль + «(внешняя ссылка)» для скринридера. Фильтры — `role="tablist"` или toggle-кнопки с `aria-pressed`. Egress-метка — текст + иконка, `aria-live` при старте сетевого прогона. Бейдж новизны — текстом, не только цвет. Прогресс обновления — `role="status"`/`aria-live="polite"`.

## 6. Поиск противоречий в базе  ⭐ приоритет 6

> Отдельный раздел/команда: пары-кандидаты по эмбеддингам/тегам → LLM-анализ типа → разрешить/объединить/архив. Дорого → только топ-кандидаты + кэш по mtime. Источник: BACKLOG, концепт §«Поиск противоречий».

### 6.1. Цель
Обнаружение конфликтующих утверждений между заметками. Три типа: **жёсткое** (X vs не-X), **мягкое** (разные акценты), **временное** (старый вывод против нового). Помочь пользователю разрешить/синтезировать/архивировать.

### 6.2. Layout / зоны
Полноэкранный раздел в оболочке 0.7 (или команда, открывающая страницу). Мастер-деталь:

```
┌ Противоречия              [↻ пересканировать]  фильтр: [Все][Жёсткие][Мягкие][Временные] ┐
│  ┌ СПИСОК ПАР ──────────┐ ┌ ДЕТАЛЬ ПАРЫ ─────────────────────────────┐ │
│  │ ● Заметка A ⇄ B  жёст.│ │  Тип: ЖЁСТКОЕ   уверенность •••          │ │
│  │ ○ Заметка C ⇄ D  мягк.│ │  ┌ A: «…утверждение…» ┐ ┌ B: «…не-X…» ┐  │ │
│  │ ○ E ⇄ F          врем.│ │  Что конфликтует (LLM, serif)            │ │
│  │ ...                   │ │  [ Разрешить ] [ Объединить ] [ Архив ]  │ │
│  └───────────────────────┘ └──────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────────────────┘
```

### 6.3. Компоненты
- **Хедер:** заголовок + «↻ пересканировать» (manual/дорого) + фильтр по типу (чипы).
- **Список пар** (левая колонка, `--color-chrome`): строки `.tree-row`-паттерн; маркер типа — точка/бейдж: жёсткое = `--color-danger`, мягкое = `--color-warning`, временное = `--color-ai` (или faint). Активная пара = `--color-selected` + 3px акцент.
- **Деталь пары** (правая, `--color-bg-elevated`, это LLM-анализ → teal-обработка/бейдж AI): 
  - заголовок «Тип: ЖЁСТКОЕ» + индикатор уверенности (3 точки);
  - две карточки-цитаты A и B бок о бок (`--color-surface`, бордюр), конфликтующие утверждения подсвечены;
  - «что конфликтует» — `font-serif` muted абзац от LLM;
  - **кнопки действий** по типу: `Разрешить` (жёсткое — открывает обе заметки + чат-синтез), `Объединить` (мягкое — создаёт заметку-синтез), `Архивировать` (временное — переносит старую в `/Archive/` с `#outdated`). Primary-кнопка по рекомендованному действию = `.sg-btn.primary` (`--color-accent`), прочие — secondary.
- **Кэш-метка:** «проанализировано N назад» (mono faint); инвалидируется при изменении одной из заметок.

### 6.4. Ключевые состояния
- **Пустое (не сканировали):** es-блок «Запустите поиск противоречий» + кнопка (это тяжёлая операция — manual).
- **Сканирование:** прогресс «этап 1: подбор пар → этап 2: LLM-анализ топ-кандидатов», анимированный бренд-марк; список наполняется по мере готовности.
- **Нет противоречий:** дружелюбный es «Противоречий не найдено» (`--color-success` оттенок).
- **Деталь не выбрана:** правая колонка — нейтральная заглушка «Выберите пару».
- **Ошибка LLM:** `.ai-banner.danger` + «повторить».
- **Из кэша:** метка времени + «обновить».

### 6.5. Взаимодействия
Клик пары → деталь. «Разрешить» открывает обе заметки в воркспейсе + чат-сессию (режим vault) для синтеза. «Объединить» создаёт новую заметку-синтез со ссылками. «Архивировать» — перенос + тег (с подтверждением, т.к. меняет файлы). Фильтр по типу. «Пересканировать» — ручной дорогой прогон.

### 6.6. Поведение в светлой/тёмной теме
Маркеры типов = статус-токены, ярче в dark (различимость типов сохраняется). Teal-обработка детали корректна обеими темами.

### 6.7. Токены
Список `--color-chrome` + `--color-selected`/`--color-accent`; типы `--color-danger`(жёст)/`--color-warning`(мягк)/`--color-ai`(врем); деталь `--color-bg-elevated` + teal-маркировка (`--color-ai`/`--color-ai-soft`, бейдж AI); цитаты `--color-surface`; действия `.sg-btn`/`.sg-btn.primary` (`--color-accent`); архив-действие может нести `--color-warning`/`--color-danger` оттенок (деструктивность); проза `--font-serif`; кэш-метка `--font-mono` faint.

### 6.8. A11y
Мастер-деталь: список — `role="list"`, навигация стрелками; выбор синхронизирует деталь (`aria-selected`). Тип противоречия — текстом + цветом (не только точка). Деструктивное «Архивировать» — подтверждение-диалог (`role="dialog"`), не одношаговое. Кнопки действий — `<button>` с понятными `aria-label`. Уверенность LLM — текстовая альтернатива точкам («высокая/средняя»).

## 7. Карта компетенций  ⭐ приоритет 7

> Месячный full-vault обзор: кластеризация → глубина → уровни словами + динамика. Автопортрет из написанного. Scheduled раз/мес. Источник: BACKLOG, концепт §«Карта компетенций».

### 7.1. Цель
Автопортрет специалиста из того, что уже написано: какие области знаний, на каком уровне (описательно), куда движутся. Честнее ручного скилл-трекера — LLM видит доказательства (объём/глубину/динамику/связность), а не самооценку.

### 7.2. Layout / зоны
Полноэкранный раздел в оболочке 0.7. Сверху — заголовок + период + дельта; центр — таблица компетенций; снизу — опц. блок наблюдений LLM; сбоку/сверху — переключатель истории по месяцам.

```
┌ Карта компетенций      [ Июнь 2026 ▾ ]   обновлено 01.06   [↻ пересобрать] ┐
│  Дельта к прошлому месяцу:  ↑ выросли 2 · ✦ новых 1 · ↓ угасает 1            │
│ ── области ─────────────────────────────────────────────────────────────  │
│  ┌ таблица ──────────────────────────────────────────────────────────┐    │
│  │ Область            │ Уровень    │ Динамика │ Последняя активность   │    │
│  │ LLM inference      │ практикую  │ ↑ растёт │ 2 дня назад            │    │
│  │ Бенчмаркинг        │ изучаю     │ → стаб.  │ 3 недели назад         │    │
│  └────────────────────────────────────────────────────────────────────┘    │
│ ── наблюдения ───────────────────────────────────────────────────────────  │
│  «Активно изучаешь inference, но почти не пишешь про бенчмаркинг — gap.» (serif) │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 7.3. Компоненты
- **Хедер:** заголовок + **селектор месяца** (dropdown — история по месяцам, архив прошлых таблиц) + статус «обновлено 01.06» (mono faint) + «↻ пересобрать» (manual, дорого — full-vault проход). Вся страница — LLM-генерируемая → teal-маркировка (бейдж AI у заголовка).
- **Дельта-строка:** чипы «↑ выросли N» (`--color-success`), «✦ новых N» (`--color-accent`), «↓ угасает N» (`--color-warning`).
- **Таблица компетенций** (4 колонки): **Область** (`--text-md`, `--color-text`); **Уровень** — описательный чип-слово *знаком / изучаю / практикую / владею* (НЕ числа; цветовая градация sage→accent по глубине, но словом); **Динамика** — иконка-стрелка ↑ (`--color-success`) / → (faint) / ↓ (`--color-warning`) + подпись; **Последняя активность** (`--font-mono`, faint). Строки — зебра через `--color-surface`/прозрачный, ховер `--color-surface-hover`. Клик по области → раскрытие/переход к заметкам кластера.
- **Блок наблюдений** (опц., под таблицей, `.h-card.ai`): `font-serif` muted, наблюдения (не советы), `em`→`--color-warning` для gap-акцента. Можно скрыть.

### 7.4. Ключевые состояния
- **Пустое (не собирали / vault мал):** es «Карта собирается раз в месяц — недостаточно данных» или CTA «Собрать сейчас».
- **Сборка (full-vault проход):** прогресс по шагам (кластеризация → оценка глубины → калибровка уровней), анимированный бренд-марк; таблица появляется готовой.
- **Готово (текущий месяц):** таблица + дельта + наблюдения.
- **Архив (прошлый месяц):** таблица того месяца, дельта скрыта/историческая, метка «архив».
- **Ошибка:** `.ai-banner.danger` + «повторить».

### 7.5. Взаимодействия
Селектор месяца листает историю (увидеть эволюцию профиля). «Пересобрать» — ручной дорогой прогон. Клик по области — к заметкам кластера/фильтр графа. Наблюдения сворачиваются.

### 7.6. Поведение в светлой/тёмной теме
На токенах. Уровень-чипы и динамика-стрелки — статус/акцент-токены, различимы обеими темами. Серьёзная серая проза наблюдений (`--color-text-muted`) держит AA на обеих.

### 7.7. Токены
Таблица: текст `--color-text`/мета `--font-mono` faint; зебра `--color-surface`; уровень-чипы — словами, цвет от `--color-tag`(знаком/изучаю)→`--color-accent`(владею) по градации (без новых цветов — берём существующие tag/accent/success); динамика `--color-success`(↑)/`--color-text-faint`(→)/`--color-warning`(↓); дельта-чипы те же; вся страница teal-маркирована (`--color-ai` бейдж/бордюр); наблюдения `.h-card.ai` + `--font-serif`, `em`→`--color-warning`; статус `--font-mono` faint.

### 7.8. A11y
Таблица — нативная `<table>` с `<th scope>`; навигация по ячейкам с клавиатуры. Уровень — слово (само по себе доступно), цвет лишь усиливает. Динамика — стрелка + текстовая подпись («растёт»), не только глиф/цвет. Селектор месяца — нативный `<select>`/доступный combobox. «Пересобрать» — `<button>` с `aria-label` + предупреждение о длительности. Наблюдения — `region` с заголовком.

## 8. Reading-mode — центрирование (рефайнмент)

> Режим чтения (⌘R) уже есть: прячет сайдбар/AI, редактор full-width. Остаётся рефайнмент — центр-measure. Источник: BACKLOG/`design.md` Ф4-9.

### 8.1. Цель
Спокойное чтение: убрать chrome, дать тексту фиксированную комфортную ширину строки по центру холста (как `--editor-measure` 760px), чтобы длина строки не зависела от ширины окна.

### 8.2. Layout / зоны
Полный холст `--color-bg`; сайдбар, AI-панель, таб-инструменты скрыты. Документ — колонка `max-width: var(--editor-measure)`, `margin: 0 auto`, вертикальные поля `--space-8` сверху / большой нижний padding. Плавающий выход (Esc / `⌘R`) — единственный chrome-элемент, ненавязчиво в углу.

### 8.3. Компоненты
- **Контент-колонка:** typography editor (`--editor-font`, `--editor-size` 16px, `--editor-leading` 1.7), проза может использовать `--font-serif`-акценты. Wikilinks `--color-link`, теги `--color-tag`-пилюли, код mono.
- **Кнопка выхода:** плавающая пилюля (`.mode-float`-паттерн: полупрозрачная `--color-bg-elevated` + blur, ховер → непрозрачна + `--color-accent` иконка). 

### 8.4. Ключевые состояния
- **Активен:** centered-колонка, chrome скрыт.
- **Выход (Esc/⌘R):** возврат к воркспейсу с прежней раскладкой.
- **Пустой документ:** центр-плейсхолдер faint.
- (Загрузка/ошибка нерелевантны — локальный контент.)

### 8.5. Взаимодействия
`⌘R` вход/выход, `Esc` выход. Ширина строки фиксирована независимо от окна; при очень узком окне (< measure) — колонка ужимается с боковыми `--space-*` отступами.

### 8.6. Поведение в светлой/тёмной теме
`--color-bg`/`--color-text` по теме; в light — «бумажное» чтение, в dark — «тёплый уголь». Ссылки/теги читаемы обеими.

### 8.7. Токены
`--color-bg`, `--color-text`, `--editor-font`/`--editor-size`/`--editor-leading`/`--editor-measure`, `--space-8`/`--space-7`, `--color-link`/`--color-tag`, `.mode-float` (`--color-bg-elevated` + `--color-accent` ховер).

### 8.8. A11y
Esc-выход обязателен и заметен. Фокус при входе — на контенте; при выходе — возврат на исходную позицию. Колонка не должна обрезать текст при зуме (reflow до 200%). Кнопка выхода — `<button aria-label="Выйти из режима чтения">`, достижима Tab.
