### Хвосты среза BF-1 (wall-clock / гейт, 2026-07-08)
- ✅ **СДЕЛАНО (2026-07-09): Конфигурируемый `wall_clock`/`max_steps`** (`ai.agent_wall_clock_secs`/`ai.agent_max_steps` в `.nexus/local.json`, дефолты 300 с / 8). Тип-толерантный парс (`de_tolerant_opt_u64` — мусор не роняет `local.json`), санитарный кламп в `AiConfig` (30..86400 с; 1..10000 шагов), резолв конфиг→границы одной точкой `LoopBounds::from_ai_config`, проброс через ПУБЛИЧНЫЙ `run_agent_session_bounded` во всех config-вызывателях (desktop-чат, agentd scheduler+коннектор, `nexus agent`, `nexus acp`); субагенты/smoke/eval — прежний дефолт. Дока: `CONFIGURATION-REFERENCE.md`.
- ✂️ **Субагент с ОБЩИМ родительским gate** (`SessionRole::Subagent { dispatcher: Some }`): решение-ожидания ребёнка кредитуются в `paused_nanos` РОДИТЕЛЯ (обёртка `PauseAccountingDecision` живёт в родительском gate), а `wall_clock` ребёнка читает СВОЙ (свежий) счётчик → ребёнок не получает вычет за собственные раздумья, а родитель получает лишний кредит. **Амплификация при КОНКУРЕНТНЫХ детях (adversarial-ревью BF-1):** N детей ждут решений одновременно, но `UiDecisionSource`/`ChannelDecision` сериализует `decide()` через `Mutex<Receiver>` → перекрывающиеся ожидания суммируются в кредит родителя ≈ N(N+1)/2·T против реального стенного N·T; кредит персистит весь прогон (счётчик не сбрасывается). Порог эксплуатации высокий: delegation И actuator — owner-gated default-OFF, решения всё равно за человеком. Направление ошибки fail-safe («родитель работает дольше», не «убит»). Триггер: живой human-in-loop делегированный прогон с длинными раздумьями по дочерним changeset'ам. Источник: BF-1 Fix 1 + adversarial-ревью.
- ✂️ **Exec в песочнице — межпроцессная пауза**: решение по exec принимается HOST-side (`dispatch_exec_decision`), а `run_agent_loop` крутится IN-CONTAINER; на время host-раздумий проксированный `dispatch` по act.sock блокирует in-container backstop-`wall_clock` (авторитетный лимит — host podman pause/kill, но in-container backstop всё же тикает). Чтобы вычесть и это, host должен репортить длительность паузы в ответе прокси, а in-container `ProxyActuator` — аккумулировать в свой `paused_nanos` (кросс-процессный контракт). Отложено как заметно больший дифф вне корня BF-1. Триггер: exec-changeset'ы с длинными host-раздумьями под коротким wall_clock. Источник: BF-1 Fix 1.
- 🐞 **PRE-EXISTING (вскрыто ревью BF-1, НЕ этот срез): `cancel_impl` не разблокирует висящий `decide()`.** Отмена прогона взводит `cancel`-флаг (проверяется на границе хода), но гейт, ЖДУЩИЙ решения (`UiDecisionSource::decide` → `rx.recv().await`), cancel не слушает → tokio-задача прогона висит на recv бессрочно (task-утечка до дропа sender'а при дерегистрации). UI цел: фронт локально ставит `cancelled` + гардит поздние события. Возможный фикс (отдельный срез): `tokio::select!` decide × cancel-notify → fail-closed `reject_all`. Источник: adversarial-ревью BF-1 (Линза-2).
- ✂️ **CLI: двойной вывод текста бюджета** (после BF-1 Fix 2): стрим печатает `✗ ошибка: бюджет исчерпан…` (render_line терминального Error-события), и `run_once` возвращает `Err(t.text)` с тем же текстом наверх. Дубль КОНСИСТЕНТЕН с pre-existing поведением provider-Error (событие + Err — та же пара и ДО BF-1); подавление одной из сторон = либо хрупкий матч по тексту события, либо смена exit-code/stderr-контракта `run_once`. Оставлено осознанно. Триггер: жалоба на шум CLI. Источник: BF-1 ревью (Линза-1 NIT).
