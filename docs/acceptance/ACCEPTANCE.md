# Критерии приёмки — Nexus

> Основа для будущих тестов (unit / integration / e2e / eval / security / perf).
> Источник истины: `ARCHITECTURE.md` (v1.1, раздел 0 — ADR). Каждый критерий имеет ID (`AC-…`) для ссылок из тест-кейсов.
> Формат: где уместно — Given/When/Then. «Порог калибруется» = значение уточняется на реальных данных, но gate обязателен.

---

## 0. Глобальные quality gates (зелёный билд)

- **AC-Q-1** `cargo build` и `pnpm build` проходят на Win/Mac/Linux без warning-as-error.
- **AC-Q-2** `cargo test` и `vitest` зелёные; покрытие критичных модулей (indexer, chunker, RRF, broker, watcher) ≥ порога (калибруется, старт 70%).
- **AC-Q-3** Линтеры (`clippy -D warnings`, eslint, tsc `--noEmit`) без ошибок.
- **AC-Q-4** RAG-eval (см. §3) не упал ниже зафиксированного baseline.
- **AC-Q-5** Все security-тесты (см. §4) зелёные — это **блокирующий** gate для merge.
- **AC-Q-6** Нет висячих упоминаний снятых решений (sqlite-vec как ANN, petgraph-источник, WASM-логика, `currentFile`) вне пояснительного контекста — линт-проверка по ключевым словам.

## 1. Критерии-доказательства закрытия блокеров Б1–Б12

### Б1 — модель плагинов (JS-first)
- **AC-Б1-1** Плагин-пример регистрирует **живое** CodeMirror-расширение (декорация/виджет) через `registerEditorExtension` и оно рендерится в редакторе. *Given* установленный JS-плагин *When* открыт документ *Then* виджет плагина виден и реагирует на изменения.
- **AC-Б1-2** Плагин получает колбэк `onFileChanged` и `AsyncIterable` от `ai.complete` без сериализационных ошибок.
- **AC-Б1-3** Матрица §7.7 покрыта тестами: UI-вью в iframe **не может** зарегистрировать редакторное расширение (получает явный отказ).

### Б2 — capability-broker и identity
- **AC-Б2-1** *Given* два плагина A (`vault:read`) и B (`vault:write`) *When* A шлёт сообщение с `pluginId:"B"` *Then* broker определяет источник по порту, отклоняет операцию записи (нет confused-deputy).
- **AC-Б2-2** Плагин с `ai:complete {local_only}` при включённом cloud-fallback: запрос уходит **только** на локальный chat-хост; попытка облака → отказ.
- **AC-Б2-3** Каждое обращение плагина к `vault.*`/`ai.*` фиксируется в audit-log (файлы, объём egress, AI-вызовы); лог неотключаем.
- **AC-Б2-4** Права path-scoped: `vault:read:["Notes/**","!Private/**"]` — чтение `Private/secret.md` отклоняется.

### Б3 — код плагинов вне git
- **AC-Б3-1** *Given* установленный плагин *When* выполняется auto-commit *Then* в коммит НЕ попадает `.nexus/plugins/**`; попадает только декларация `id@version#sha256` в `config.json`.
- **AC-Б3-2** *Given* pull принёс новый/изменённый плагин *Then* плагин в состоянии `disabled, needs-review`, требует явного consent на этом устройстве.
- **AC-Б3-3** Установка из marketplace проверяет подпись и sha256; повреждённый/неподписанный артефакт отклоняется (с Фазы 2).

### Б4 — эмбеддинг по чанкам
- **AC-Б4-1** *Given* файл из 3 разнотемных секций *When* индексация *Then* в usearch ≥3 вектора с различающимися значениями (не один файловый вектор).
- **AC-Б4-2** Число векторов файла = числу его чанков; при реиндексации старые векторы файла удалены (нет роста-дублей).

### Б5 — размерность эмбеддера
- **AC-Б5-1** Размерность usearch-индекса = `embedder.dim()` (не хардкод); запись вектора иной длины отклоняется.
- **AC-Б5-2** *Given* смена `embedding.model` (768→1024) на старте *Then* индекс пересоздан, файлы помечены к переэмбеддизации, старые векторы не смешиваются (§6.5).

### Б6 — реальный ANN
- **AC-Б6-1** Семантический поиск использует usearch HNSW; на 500k чанков латентность ≤ порога (см. AC-PERF-3), а не линейный рост.
- **AC-Б6-2** Префильтр по метаданным (папка/тег/дата) применяется ДО KNN (проверяется планом запроса/счётчиком кандидатов).

### Б7 — БД-слой
- **AC-Б7-1** Запись идёт через единый write-actor; конкурентные реиндексации не дают `SQLITE_BUSY` (стресс-тест N параллельных изменений).
- **AC-Б7-2** Транзакция индексации атомарна: сбой на середине не оставляет файл с частью таблиц (files без chunks и т.п.).

### Б8 — FTS5 и чистка векторов
- **AC-Б8-1** FTS-поиск находит текст заметки сразу после сохранения (триггеры синхронизации работают).
- **AC-Б8-2** *Given* удаление файла *Then* его строки в `fts_chunks` и векторы в usearch удалены (нет «призраков» в выдаче).

### Б9 — atomic-save и watcher
- **AC-Б9-1** *Given* заметка с N беклинками *When* сохранение через tmp+rename (как делают редакторы) *Then* `file_id` НЕ изменился, `links/chunks/vec/tags` НЕ удалялись каскадно, беклинки целы.
- **AC-Б9-2** Запись в `.nexus/nexus.db*` НЕ порождает событие индексации (watcher `is_ignored`); нет цикла реиндексации.
- **AC-Б9-3** Шторм из M событий за окно дебаунса схлопывается; пара remove+create трактуется как один upsert по пути.

### Б10 — стриминг чата
- **AC-Б10-1** *Given* две параллельные сессии чата *Then* токены не смешиваются (каждый `Channel` пишет в свой буфер).
- **AC-Б10-2** На событие `done` ответ ассистента финализируется в историю (`chat_messages`) и виден после перезагрузки.
- **AC-Б10-3** `cancel` останавливает генерацию на сервере (нет «фоновой» траты GPU); UI переходит в `idle`.
- **AC-Б10-4** Рендер ответа на 2000 токенов не деградирует (throttled-снапшоты; нет O(n²) ре-рендеров — бюджет кадра соблюдён).

### Б11 — изоляция WASM и пути
- **AC-Б11-1** `vault.readFile("../../.ssh/id_rsa")` и абсолютный путь вне vault → отказ `PathEscape`; симлинк наружу → отказ.
- **AC-Б11-2** WASM busy-loop прерывается по epoch ≤ 5s; процесс приложения жив.
- **AC-Б11-3** Превышение 256MB в WASM → отказ resource-limiter, без падения процесса-хоста.

### Б12 — вкладки/сплиты
- **AC-Б12-1** Открыты ≥2 вкладки и ≥2 сплита; AI-контекст/backlinks берутся из активной вкладки активной группы (не из глобального файла).
- **AC-Б12-2** Грязный буфер сохраняет «dirty» при переключении вкладок; данные не теряются.

## 2. RAG-eval (раздел 6.6) — блокирующий gate качества

- **AC-EVAL-1** Существует golden-набор `вопрос → ожидаемые чанки/файлы` по реальному vault (≥ N кейсов, калибруется).
- **AC-EVAL-2** Метрики на golden: **recall@8 ≥ порог**, **nDCG@8 ≥ порог**, **MRR ≥ порог** (пороги фиксируются как baseline в Фазе 1).
- **AC-EVAL-3** Любое изменение чанкера/RRF-весов/порогов/модели прогоняет eval; merge запрещён при регрессии метрик ниже baseline.
- **AC-EVAL-4** Сравнения валидны только при совпадении условий (модель, сервер, набор) — фиксируется в отчёте прогона.
- **AC-EVAL-5** Порог similarity для link-suggest откалиброван per-model (не зашитый 0.75).
- **AC-EVAL-6** Кросс-язычный кейс: запрос на RU по EN-заметке (и наоборот) попадает в recall@8 (проверка мультиязычного эмбеддера, ADR-005).

## 3. Безопасность (раздел 11, «Модель угроз») — блокирующий gate

- **AC-SEC-1** Path-traversal/симлинк-побег заблокированы во ВСЕХ host-функциях и Tauri-командах (`resolve_vault_path`).
- **AC-SEC-2** Capability-laundering предотвращён: плагин не может вынести vault в облако (AC-Б2-2) и не может выдать себя за другого (AC-Б2-1).
- **AC-SEC-3** Исполняемый код плагина не доставляется через git (AC-Б3-1); pull-приход → needs-review (AC-Б3-2).
- **AC-SEC-4** SSRF: `*.url` с приватным/metadata-адресом (`169.254.169.254`, `192.168.*`, внутренние) отклоняются без явного согласия; изменённый из pull `base_url` не применяется без подтверждения. **Исключение ядра (E3, ADR-005-ext):** явно сконфигурированные пользователем `ai.chat`/`ai.embedding` url — включая LAN (`192.168.*`/loopback) — разрешены ядру by design (local-first; не приходят из pull без подтверждения, см. выше); **metadata `169.254.169.254` — reject ВСЕГДА** (отдельный предикат). Плагинный `net.fetch` остаётся под полным `is_private_host` без исключений.
- **AC-SEC-5** Tauri-команды `vault.*`/`git.*` недостижимы из iframe-контекста плагина (только через broker); CSP без `unsafe-inline`/`unsafe-eval`.
- **AC-SEC-6** Секреты не в логах: контент заметок и пути не печатаются (тип `Redacted`); crash-reporter скрабит пути, строго opt-in.
- **AC-SEC-7** Prompt-injection: заметка с `</note>`/«ignore previous instructions» не ломает контекст и не управляет suggest (разделители + валидация JSON-ответа).
- **AC-SEC-8** (опц.) При включённом at-rest шифровании `nexus.db` нечитаема без ключа из keychain.

## 4. Производительность (раздел 10) — пороги pass/fail

> Условия замера фиксируются: vault 50k файлов (≈150–400k чанков), референс-железо (указать в CI).

- **AC-PERF-1** UI интерактивен (редактор виден и принимает ввод) < 1 с; индекс/граф/usearch грузятся в фоне, не блокируя UI.
- **AC-PERF-2** Открытие файла < 50 мс; FTS-поиск < 100 мс.
- **AC-PERF-3** Семантический поиск (usearch + префильтр) < 300 мс на полном индексе.
- **AC-PERF-4** RAG до первого токена < 1 с.
- **AC-PERF-5** Индексация: измеренный throughput эмбеддера зафиксирован; прогресс «N/M чанков»; полная индексация **резюмируема** после краша (reconcile по `indexed_at`).
- **AC-PERF-6** Граф: layout считается в Web Worker и не блокирует main-thread > 1 кадра; локальный N-hop открывается < 1 с.
- **AC-PERF-7** Файловое дерево 50k: скролл без джанка (виртуализация flatten-узлов), ленивые дети.

## 5. i18n (раздел 9)

- **AC-I18N-1** Нет недостающих ключей между ru/en (CI-линтер).
- **AC-I18N-2** Русские плюралы корректны: «1 файл / 2 файла / 5 файлов» (категории one/few/many).
- **AC-I18N-3** Числа/даты форматируются через `Intl` под активную локаль (50000 → «50 000»).
- **AC-I18N-4** Сортировка дерева/списков через `Intl.Collator('ru')` (Ё, регистр, числовой порядок).
- **AC-I18N-5** Системная локаль определяется при первом запуске; смена языка применяется без потери данных.
- **AC-I18N-6** Ошибки бэкенда (Rust: индексатор/git/AI) локализованы, не только en.
- **AC-I18N-7** Плагин поставляет переводы в namespace `plugin:<id>:<key>`; UI плагина локализуется.

## 6. Definition of Done по фазам (раздел 12)

- **AC-DOD-Ф0** Открывается существующий Obsidian-vault; работают вкладки/сплиты, source-mode редактор, клик/автокомплит `[[wikilink]]`, `#tags`, FTS, беклинки из SQLite, command-registry, i18n RU/EN; rusqlite+миграции; watcher с reconcile (AC-Б9-*); CSP/capabilities-аудит. **Live Preview НЕ требуется в Ф0.**
- **AC-DOD-Ф1** «Что я знаю о X?» возвращает ответ с источниками; чанк-эмбеддинги (AC-Б4), hybrid+RRF без +0.2, стриминг через Channel (AC-Б10), suggest режим 1 (max-sim), eval-харнесс с baseline (AC-EVAL-*), мультиязычный эмбеддер, cloud-fallback opt-in.
- **AC-DOD-Ф2** Сторонний разработчик пишет JS-плагин по докам и SDK; broker+scoped-permissions+audit (AC-Б2), подпись+реестр (AC-Б3-3), lifecycle install/update/rollback/uninstall, матрица §7.7 соблюдена.
- **AC-DOD-Ф3** Vault 50k: AC-PERF-* выполнены; git-sync core-module (выборочный коммит+secret-scan, sync-lock, AC-Б3-1, AC-SEC-3); единый граф + worker-layout.
- **AC-DOD-Ф4** Тема light/dark (+ протекание в iframe-плагины), просмотр вложений/не-md, опц. at-rest шифрование, crash-reporter со скрабингом (AC-SEC-6), auto-updater с подписью, onboarding.

## 7. Definition of Done для PR / компонента

- **AC-PR-1** Изменение покрыто тестом(ами) с ссылкой на соответствующий `AC-…`.
- **AC-PR-2** Затронут публичный Plugin API → обновлены типы SDK, `min_api_version`/capabilities, changelog; CM6-пин не сломан.
- **AC-PR-3** Затронута схема БД → миграция + тест миграции (вкл. пересоздание usearch при смене размерности).
- **AC-PR-4** Затронут UI → состояния empty/loading/streaming/error/offline реализованы; a11y (фокус/ARIA/контраст); i18n-ключи добавлены в ru и en.
- **AC-PR-5** Затронут host↔plugin путь или конфиг → добавлен/обновлён security-тест (traversal/identity/SSRF/egress).
- **AC-PR-6** Нет регрессии eval (AC-EVAL-3) и perf-порогов (AC-PERF-*).

## Допущения и placeholder’ы

- Числовые пороги eval (recall/nDCG/MRR), порог similarity, покрытие, и точные perf-числа на конкретном железе — **placeholder**, фиксируются как baseline в Фазе 1 на реальном vault.
- Размер golden-набора (N кейсов) и состав референс-железа для perf-замеров — определяются в начале Фазы 1/3.
- Список «приватных/секретных» папок для path-scoping — конфигурируется пользователем; тест проверяет механизм, не конкретные пути.
