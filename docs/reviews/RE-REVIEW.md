## Новые противоречия / регрессии — найдено и устранено

Правки v1.0→v1.1 оставили висячие упоминания старого стека; найдены и исправлены в ходе ре-ревью:

- §6.1 commit-box: `- vec_chunks` → `- usearch ANN`; embedder-box → `embedding_url`/per-chunk/отд. хост.
- §6.2 hybrid-box: `sqlite-vec ANN` → `usearch HNSW`.
- §10 метрика старта: `sqlite-vec-загрузка` → `usearch-mmap`.
- §5 структура vault: `nexus.db-vec` → `vectors.usearch`; добавлены `local.json` и пометка «плагины не в git».
- §2 first-party: граф и git-sync помечены как **core modules** (не sandbox-плагины).
- §4.4: ссылка на матрицу `§7.6` → `§7.7`.

После правок повторный grep на `sqlite-vec / petgraph / WASM-песочница / currentFile / FLOAT[1024] / add_all(["*"]) / llm-stream` даёт совпадения **только в пояснительном контексте** («что заменено / чего НЕ делать»). Code-блоки сбалансированы (44 пары), все разделы 0–13 + приложения на месте.

## Остаточные оговорки (НЕ блокеры — для фазы реализации)

1. **Уровень детализации.** Ряд фиксов даны на уровне сигнатур/комментариев, а не полного кода: протокол broker (wire-format, capability-токены), точная reconcile-логика watcher, фактический энфорсмент WASM-лимитов. Для арх-документа это нормально, но это первые кандидаты на детализацию в дизайне Фаз 0/2.
2. **§1, таблица принципов.** Формулировка «Plugin-first: каждая необязательная фича — плагин» слегка в напряжении с категорией core modules; нюанс раскрыт в §7/§13, но саму строку принципа не трогали.
3. **§7.3 SDK-код** оставлен с исходной поверхностью API (она валидна для JS-first), но проводка capability-токенов/портов показана отдельно в §7.4/§7.5, а не в самом листинге SDK.

## Вердикт ре-ревью

Все 12 блокеров закрыты архитектурно; после устранения шести висячих упоминаний документ внутренне непротиворечив и согласован с разделом 0 (ADR). Остаточные пункты — это работа уровня реализации/дизайна фазы, а не архитектурные дыры. Документ прошёл повторную проверку.

## Обновление (та же сессия): остаточные оговорки добиты

После ре-ревью design-level пробелы из «Остаточных оговорок» закрыты прямо в документе:
- **§7.9** — протокол broker↔плагин: wire-format конверта, capability-токены, схема audit-log, изоляция/таймаут/отмена, версия протокола. (Закрывает оговорку #1 в части broker.)
- **§4.2** — reconcile-алгоритм watcher (Upsert/Rename/Delete по пути, сохранение `file_id`), echo-suppression, нормализация путей и резолвинг wikilink, изоляция паник/отмена на IPC-границе. (Оговорка #1 в части reconcile; gap нормализации путей.)
- **§5.1** — миграции схемы (вкл. пересоздание FTS5/usearch) и восстановление после краха (reconcile-проход по `indexed_at`/hash, персист-очередь).
- **§4.6** — Command Registry + клавиатурная модель (ядро + плагины, разрешение конфликтов).
- **§1** — формулировка принципа Plugin-first уточнена под категорию core modules. (Оговорка #2.)
- **§7.3** — ссылки на протокол §7.9 и матрицу §7.7. (Оговорка #3.)

Остаётся истинно implementation-level (не дизайнится на бумаге): фактический энфорсмент WASM-лимитов (epoch/fuel/StoreLimits — код Фазы 2) и эмпирическая калибровка порогов eval/perf (Фазы 1/3). Это не пробелы документа.
