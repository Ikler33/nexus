# Продуктовый аудит Nexus (2026-06-13)

> Глубокий многоагентный аудит (ultracode): 10 параллельных полос анализа по реальному коду → синтез 
> → адверсариальный критик полноты. 12 агентов, ~1.6M токенов. Цель — «второй мозг, которым пользуются 
> КАЖДЫЙ ДЕНЬ». Каждая находка обоснована файлом/поведением и сравнена с лучшими приложениями мира.

> План реализации по итогам — `docs/IMPROVEMENT_PLAN.md`.


## Резюме

Nexus уже сильнее всего как «AI-мыслитель поверх vault»: промышленный бэкенд данных (SQLite WAL + single-writer-actor, версионированные миграции, краш-устойчивый планировщик с dead-видимостью и супервизором, инкрементальный индексатор + reconcile после краха), мощный, но недоиспользованный гибридный RAG (вектор+FTS5+граф+RRF+реранк, eval-гейт), чат с цитатами и памятью сессий, inline-LLM (continue/rewrite/summarize) прямо в редакторе, проактивные Home-инсайты (open questions, context drift, stale radar, противоречия), беклинки, граф, News Feed, git-sync с secret-scan, 4 темы на OKLCH, грамотный i18n RU/EN, аккуратный motion-слой. Это отличная основа «думать и связывать».

Но для цели «второй мозг КАЖДЫЙ ДЕНЬ» приложение пока «открываешь осознанно», а не «рефлекторно». Три структурных провала держат его на этом уровне: (1) ЗАХВАТ слишком медленный — quick-capture и daily note существуют лишь как две кнопки на Home без команды/хоткея/глобального окна, новая заметка всегда пустой Untitled.md в корне без фокуса курсора, нет шаблонов; (2) ВОЗВРАТ к знаниям сломан — дорогой контентный/семантический поиск НЕ подключён к UI (ищем только по заголовкам), нет quick-switcher недавних, нет навигации назад/вперёд, клик по тегу даёт зашумлённую подстроку, ресёрфейсинга прошлого (on-this-day/случайная) нет совсем; (3) СОХРАННОСТЬ под угрозой — нет автосейва (правки живут в RAM до Ctrl-S), закрытие окна/вкладки не флашит, внешнее изменение молча затирает буфер, запись файла не атомарна, нет истории версий/корзины. Эти три темы и должны определять дорожную карту.

5 главных рычагов ежедневности: (1) Автосейв + защита данных — без этого доверять второму мозгу нельзя. (2) Глобальный quick-capture (хоткей+оверлей, append с таймстампом) + daily/capture как команды с биндами. (3) Подключить контентный поиск searchContent в сайдбар и палитру (бесплатный движок уже готов). (4) Ежедневный цикл: quick-switcher недавних (⌘O), back/forward, фокус курсора при создании, шаблоны daily. (5) Ресёрфейсинг «на этот день»/случайная + проактивные Home-инсайты по умолчанию (recurring, без «жми refresh»). Бонус-дифференциатор: связать инсайты и цитаты с действиями в ИИ (мост «увидел → разобрал с ассистентом»).


## Сквозные темы


### ▸ Захват слишком медленный — сердце второго мозга недоделано

Daily note и quick-capture существуют лишь как две кнопки на Home без команды/хоткея/глобального окна; quickThought открывает Inbox.md без append и без фокуса; новая заметка — пустой Untitled.md в корне без курсора и без title-first; шаблонов нет; глобального системного захвата нет (grep global_shortcut/tray пуст). «Мысль → в систему за <1 сек из любого места» сейчас занимает: уйти на Home, найти кнопку, проскроллить, поставить курсор, печатать. Через 2 недели захват умирает. Чинит: глобальный quick-capture-оверлей, daily/capture как команды с биндами, append+таймстамп, фокус курсора, шаблоны daily.


### ▸ Возврат к знаниям сломан — помнишь мысль, а не имя файла

Дорогой гибридный searchContent (вектор+FTS5+граф+RRF+сниппеты) имеет 0 вызовов в UI — ищем только по заголовкам через substring searchVault. Нет quick-switcher недавних (⌘O занят выбором папки), нет навигации назад/вперёд, нет fuzzy в палитре, клик по тегу даёт зашумлённую подстроку вместо точного среза, результаты без сниппета/подсветки, ресёрфейсинга прошлого (on-this-day/случайная/flashback) нет совсем — Home смотрит только вперёд. Половина накопленной глубины базы мертва. Чинит: подключить searchContent, recents+back/forward, точный tag-фильтр, on-this-day, fuzzy-палитра.


### ▸ Сохранность данных под угрозой — доверять второму мозгу пока рискованно

Нет автосейва (правки в RAM до Ctrl-S), закрытие окна/вкладки не флашит и не предупреждает, внешнее изменение молча затирает dirty-буфер, write_file не атомарен, git pull делает force-checkout, нет истории версий/корзины, нет rename/delete из UI. Каждый пункт — потенциальная тихая потеря годами накопленных мыслей. Для инструмента «каждый день постоянно» это подрывает фундамент доверия сильнее любого UX-бага. Бэкенд БД при этом крепкий — провал именно на письменном пути фронта. Чинит: автосейв-дебаунс+flush, onCloseRequested, баннер внешнего изменения, atomic write, commit-then-pull, .nexus/history, корзина.


### ▸ Проактивность есть, но обрывается на показе — нет моста инсайт→действие

ИИ САМ находит open questions, context drift, противоречия, связи, похожие — это сильная сторона. Но: ключевые AI-карточки пусты по умолчанию и требуют «жми refresh»; клик по инсайту ведёт только в файл, нет «разобрать с ИИ»; цитаты [n] в ответе не кликабельны; у ответа нет действий (копировать/повторить/в заметку); активная заметка почти невидима чату; «причина связи» — сырой сниппет, а не объяснение; живая сводка размышления выбрасывается. Ежедневный ритм «увидел инсайт → одним кликом работаю над ним с ассистентом» отсутствует. Чинит: recurring-инсайты, кнопки «Разобрать с ИИ», кликабельные цитаты, action-row на ответе, контекст активной заметки.


### ▸ Ежедневный цикл и временная ось отсутствуют как концепция

Нет daily notes как якоря дня, нет задач/todo первым классом (чекбоксы мёртвые), нет шаблонов, нет календаря/планнера, нет ресёрфейсинга, нет spaced-repetition. Daily brief — технический diff, а не «вот твой день»; цели read-only без жизненного цикла; streak/heatmap по mtime врут при синке. Эти примитивы — то, что превращает «папку файлов» в дневник, дающий причину возвращаться даже в дни без новых заметок. Это стратегические ставки (L/XL), определяющие, станет ли Nexus ежедневным.


### ▸ Полировка и обнаруживаемость — близко к Linear/Things, но не дотянуто

Солидный фундамент (motion-слой, 4 OKLCH-темы, reduced-motion, i18n RU/EN, клавиатура в FileTree/палитре), но: нет глобальной toast-системы и undo, фокус-трап в модалках отсутствует при aria-modal=true, Escape закрывает не все диалоги, нет шпаргалки хоткеев (?/⌘/), палитра substring а не fuzzy без категорий/recent, свежий vault пуст (welcome не засевается, приземление на стену пустых Home-карточек), стартовые pills чата про сам Nexus а не про vault, английские строки в русском Home. Каждая мелочь по отдельности невелика, но вместе формируют «продумано до мелочей» — то, ради чего возвращаются ежедневно.


## Топ-баги (чинить в первую очередь)

- **Нет автосейва — правки живут только в памяти до ручного Ctrl-S; краш/kill/закрытие теряет текст** _(Производительность, надёжность, сохранность данных)_ — Подтверждено в коде: updateBufferDoc (workspace.ts:173) лишь ставит dirty=true в zustand; saveBuffer (181) вызывается ТОЛЬКО из Mod-s и команды палитры. Второй мозг обязан НИКОГДА не терять мысль — пользователь Notion/Reflect/Apple Notes не жмёт Ctrl-S. Фикс дешёвый: debounced-autosave 800-1500мс + flush на blur/смену вкладки/потерю фокуса окна; Mod-s оставить как немедленный flush. Это фундамент доверия.
- **Закрытие окна/вкладки не флашит и не предупреждает о несохранённом** _(Производительность, надёжность, сохранность данных)_ — Нет ни одного onCloseRequested/beforeunload (grep пуст — подтверждено); closeTab (workspace.ts:106) GC-ит буфер без записи на диск. Рефлекторное Cmd-W/Cmd-Q = тихая потеря. Подписаться на Tauri onCloseRequested и флашить dirty-буферы; в closeTab флашить перед GC. Парный фикс к автосейву.
- **Внешнее изменение файла молча затирает несохранённый dirty-буфер в редакторе** _(Производительность, надёжность, сохранность данных)_ — Editor.tsx:126-140 при watcher-reload/git pull заменяет весь doc без проверки dirty. В git-синкаемом/мультиустройственном vault это ежедневная потеря активной мысли. Нужен баннер «Файл изменён на диске» (Оставить мои / Загрузить с диска / Сравнить) по расхождению хеша — как в VS Code.
- **git pull/resolve делают force-checkout, рискуя снести несохранённые буферы и незакоммиченные правки** _(Производительность, надёжность, сохранность данных)_ — git/mod.rs:411-412 и :521-522 вызывают checkout_head(...force()); git_sync делает pull без проверки чистоты дерева и без флаша буферов. «Нажал Синхронизировать → потерял правку» — худший вид потери. Перед pull: флашить буферы + commit-then-pull, никогда не force поверх незакоммиченного.
- **Запись файла не атомарна (tokio::fs::write напрямую) — краш в момент записи рвёт .md** _(Производительность, надёжность, сохранность данных)_ — vault.rs:629 — прямой tokio::fs::write (truncate+write). Иронично: watcher уже умеет atomic-save tmp→rename, но сам write её не делает. Заметка — единственный источник истины (БД лишь индекс). Писать через временный файл + атомарный rename (стандарт). Effort S.
- **Редактор не получает фокус при создании/открытии заметки — печатать нельзя без клика мышью** _(Захват и ежедневный поток)_ — Editor.tsx создаёт EditorView и при смене файла делает dispatch с anchor, но НИГДЕ не зовёт view.focus(). После createNote→openFile курсор не стоит. Микро-трение после каждой новой заметки накапливается в «лень записать». Добавить view.focus()+курсор в конец в эффект монтирования/смены файла. Effort S.
- **Клик по тегу делает substring-поиск имени, а не точную фильтрацию по тегу** _(Поиск, навигация, возврат к знаниям)_ — Sidebar.tsx:71 searchByTag → setQuery(name) → search_notes LIKE %name% по path/title/tag. Тег #project подтянет Projects/Roadmap.md по пути и «My project» по заголовку без тега. Точный префильтр tag= в гибриде уже поддержан, но не используется. Зашумлённый срез убивает доверие к тегам. Effort S.
- **Клик по [[wikilink]] на несуществующую заметку — молчаливый no-op (не создаёт заметку)** _(Редактор и письмо)_ — workspace.ts:88 openLink: resolveNote(...).catch(()=>null); if(path) openFile — если заметки нет, ничего не происходит. «Писать [[Идея]] до её создания и кликом материализовать» — главный механизм роста графа знаний; сейчас это тупик. Создавать заметку при null + рисовать висячие ссылки приглушённо (как Obsidian). Effort S.
- **Ошибка ответа чата — тупик без «Повторить»** _(Полировка / AI-напарник)_ — ChatView.tsx:366 при error рендерит статичный <p> без retry; ключа chat.retry нет. LLM на 192.168.0.31 может моргнуть (таймаут/перезагрузка рига) — самый частый сбой. Без одношагового повтора последний вопрос надо вводить заново = потеря мысли. Добавить inline «Повторить»/«Копировать вопрос». Effort S.
- **Вложенные vault-картинки не рендерятся в preview (convertFileSrc не применён)** _(Редактор и письмо)_ — MarkdownPreview img идёт через дефолтный react-markdown; convertFileSrc (asset:-протокол) используется ТОЛЬКО в FileViewer. Относительный attachments/x.png в вебвью не резолвится → битая картинка. Скриншоты/фото — базовый материал второго мозга; их невидимость подрывает доверие к режиму чтения. Обернуть относительный src через convertFileSrc. Effort S.


## Quick wins (S-усилие, высокий эффект)

- **Подключить searchContent (полнотекст+семантика) в сайдбар и палитру** `[high/S]` _(Поиск, навигация, возврат к знаниям)_ — Подтверждено: searchContent имеет НОЛЬ вызовов в реальных компонентах — Sidebar.tsx:42 и CommandPalette.tsx:57 зовут только searchVault (substring по заголовкам). Дорогой RAG-движок (вектор+FTS5+граф+RRF+сниппеты) простаивает вне чата. «Ты помнишь МЫСЛЬ, а не имя файла» — сердце возврата. Сниппет уже в SearchHit.snippet, бэкенд готов: переключатель «Заголовки/Везде» в сайдбаре + секция «По содержимому» в палитре. Огромный эффект за S.
- **Зарегистрировать daily-note и quick-capture как команды с хоткеями (mod+shift+d / mod+shift+n)** `[high/S]` _(Захват и ежедневный поток)_ — Логика dailyNote()/quickThought() уже есть в HomeView, но НЕ зарегистрирована в commands-core.ts — доступна только кнопкой на Home. Вынести в lib/daily.ts, зарегистрировать 2 команды → появятся в палитре и получат бинды. Ежедневный якорь (today) и захват должны вызываться из любого экрана. Дёшево, разблокирует ритуал.
- **Quick-capture: append с таймстампом + автофокус, вместо open+overwrite** `[high/S]` _(Захват и ежедневный поток)_ — quickThought() сейчас просто открывает Inbox.md (тело '# Inbox'), без добавления строки и без фокуса курсора (anchor=0). Минимальный шаг до полноценного модала: append строкой `- HH:MM <текст>` + тост «Захвачено» + курсор в конец. Превращает мёртвую кнопку в реальный захват. (Глобальный оверлей — отдельная high-impact ставка.)
- **Проактивные Home-инсайты по умолчанию: open_questions в recurring + seed на открытии** `[high/S]` _(Home, обзор и ресёрфейсинг)_ — commands/vault.rs:144 — open_questions и слой2 stale MANUAL-only; пусты до клика «refresh», empty-стейт буквально просит «hit refresh». Самые мыслестимулирующие карточки де-факто выключены. Перевести open_questions в recurring + run-if-overdue seed (как уже сделано для daily_brief/context_drift). На Home ничего не должно ждать клика — ассистент сам всё подготовил.
- **Quick-switcher недавних файлов на ⌘O (recents-стор + пустой запрос палитры)** `[high/S]` _(Поиск, навигация, возврат к знаниям)_ — ⌘O сейчас = vault.open (выбор папки), истории открытия нет, при пустом запросе палитра показывает только команды. 80% навигации второго мозга — «вернись к тому, над чем только что работал». Кольцевой буфер N путей (писать в openFile, персист localStorage) + секция «Недавние» в палитре + ⌘O в режим файлов. Obsidian/VS Code-паттерн.
- **Форматные хоткеи ⌘B/⌘I/⌘K (и снять ⌘I с inline-LLM)** `[med/S]` _(Редактор и письмо)_ — Editor.tsx биндит Mod-i на inline-continue, ⌘B/⌘K отсутствуют. ⌘B/⌘I/⌘K — мышечная память любого пишущего; их нет → вручную печатаешь ** и [](). Захват ⌘I под LLM ломает ожидание italic. wrapSelection-хелпер + перенос LLM-continue на Mod-j/Mod-Shift-Space. Дёшево, ежедневный комфорт письма.
- **Карточка «На этот день» + кнопка «Случайная заметка» на Home** `[high/M]` _(Home, обзор и ресёрфейсинг)_ — grep подтвердил: 0 совпадений по on-this-day/random/resurface. Вся ось Home направлена вперёд; единственный взгляд назад — Stale radar в негативной рамке «устарело, почини». Ценность базы — во всплытии забытого хорошего. created_at уже есть в files; read-only SQL (ORDER BY RANDOM, дата-матч), мгновенно, офлайн. Тёплая рамка «год назад ты писал…». (Effort между S и M — но эффект на возвращаемость высокий.)

Лента обрывается на 50-й, старое стирается 30-дневной ретенцией → потеря знания. page/hasMore в сторе + IntersectionObserver на сентинеле. Сервер готов, чистый фронт.


## High-impact (M/L)

- **Глобальный quick-capture: системный хоткей → always-on-top окно → append в Inbox/today** `[high/M]` _(Захват и ежедневный поток)_ — grep по global_shortcut/tray пуст — все точки захвата внутри окна. Самые ценные мысли приходят, когда Nexus не на переднем плане. Tauri global-shortcut → минимальное окно → Enter дописывает в Inbox.md с таймстампом и прячется, не теряя контекста редактора. Это ПРИЧИНА, по которой Reflect/Mem/Drafts/Raycast используют ежедневно. Эффект — превращает Nexus из «открываю осознанно» в «рефлекторно». Уважать egress (только запись в vault).
- **Контекст активной заметки в чат + «Спросить ИИ о выделенном»** `[high/M]` _(AI как ежедневный напарник)_ — ChatView.tsx:71 передаёт только путь в center → 3-й источник граф-ранга; содержимое активной заметки в промпт НЕ инжектится напрямую, конкурирует с vault на общих основаниях. Главный жест второго мозга — «смотрю на заметку и хочу про НЕЁ поговорить». Чип «Эта заметка» (pin чанков как primary) + действие «Спросить про выделенное» из редактора + empty-state «О чём этот файл?». Замыкает «то что читаю ↔ то что спрашиваю».
- **Действия на ответе ассистента: копировать / повторить / в заметку / редактировать вопрос** `[high/M]` _(AI как ежедневный напарник)_ — В Message() рендерятся только content+источники; grep copy/regenerate/retry/edit пуст. Экспорт «в заметку» есть только на ЦЕЛУЮ сессию. Базовый цикл доверия: получил → скопировал кусок в заметку, перегенерил неудачный, поправил вопрос. content уже в сторе — технически просто. Без этого хороший ответ застревает в чате.
- **Глобальная toast-система с опциональным action (Открыть/Отменить)** `[high/M]` _(Полировка)_ — Единственный тост — локальный в NewsView без анимации; chat→note показывает галочку и .catch(()=>{}) ГЛОТАЕТ ошибку молча, путь не показывается. Второй мозг — доверие к сохранению: «готово, вот куда легло» + ссылка открыть. Единый toast-стор + ToastViewport(aria-live), перевести news/chat→note/capture/реиндекс/sync. Фундамент под undo и под видимость провалов сохранения.
- **Шаблоны заметок + богатый daily-шаблон (папка Templates/, плейсхолдеры даты)** `[high/M]` _(Захват / Редактор / Стратегия)_ — Шаблонов нет совсем; dailyNote() пишет фиксированное '# date\n\n'. Папка Templates/ с {{date}}/{{time}}/{{weekday}} + выбор из палитры; минимум — дать daily богатый дефолт с переопределением из Templates/daily.md. Разблокирует ежедневный ритуал и сравнимость заметок. Vision-уровень (LLM-«умный шаблон» из похожих заметок) — естественный дифференциатор Nexus.
- **Навигация назад/вперёд по истории переходов (⌘[ / ⌘])** `[high/M]` _(Поиск, навигация, возврат к знаниям)_ — Стека истории навигации нет, команд back/forward нет. Мышление во втором мозге — блуждание по ссылкам (заметка→беклинк→граф→обратно); без back теряешь нить и боишься углубляться. Push при openFile/openLink + nav.back/forward + mouse back-кнопка. Базовый рефлекс из браузеров, must-have для link-navigation.
- **Live Preview оплата частями: вставка/paste изображений + интерактивные чекбоксы + рендер картинок** `[high/M]` _(Редактор и письмо)_ — Нет paste/drop изображений (grep пуст) — вставить скриншот ⌘V невозможно, надо вручную копировать файл и писать путь. Чекбоксы в preview нерабочие (read-only). Это рефлексы ежедневного захвата и daily-практики. CM6 domEventHandlers paste/drop → сохранить в attachments/ + вставить ![[]]; чекбоксы — маппинг node.position в исходник + патч [ ]↔[x]. Промежуточные победы до полного Live Preview (XL).
- **Мост «инсайт → разобрать с ИИ» на Home-карточках и кликабельные цитаты [n]** `[high/M]` _(AI как ежедневный напарник)_ — Open Questions/Context Drift/противоречия — тупиковые: клик ведёт только в файл, нет действий. Проактивность (ИИ САМ находит вопросы/дрейф — это сильно!) обрывается на показе. Кнопка «Разобрать с ИИ» (openChat + префилл send с center=path). - **Локальная история версий заметок (.nexus/history снапшоты + diff/restore)** `[high/M]` _(Производительность, надёжность, сохранность данных)_ — grep backup/snapshot/version/trash пуст; единственная защита — ручной git-sync (его большинство не настроят). Капчур без курации = свалка; «Untitled» не переименовать, welcome-заметка обещает удаление, которого нет. rename-with-backlink-update (киллер-фича Obsidian) + контекстное меню FileTree + удаление в системную корзину, не hard-rm.


## Стратегические ставки (L/XL)


### Полноценный ежедневный цикл: daily notes как якорь + календарь + проактивный утренний бриф

**Обоснование:** Date-индекса заметок нет; daily/inbox — тончайшие заглушки без шаблона, навигации вчера/сегодня/завтра, on-this-day. Daily brief сейчас — технический diff изменённых файлов («что ты И ТАК знаешь»), а не «вот твой день». Это разблокирует временную ось мышления и даёт LLM сильнейший контекст «что я думал в этот период». Цель: команда «Open today» → Journal/YYYY-MM-DD.md по шаблону с prev/next, виджет «Сегодня» как живой редактируемый блок на Home, capture по умолчанию летит в today, мини-календарь в сайдбаре (клик по дню → daily), бриф переписать как синтез open-questions + stale + due + on-this-day в один нарратив. Heatmap/streak перевести на реальную таблицу edit_events (сейчас по files.updated_at → врёт при ребилде/синке).

**Эталон:** Roam/Logseq — daily note как центр всей системы, всё начинается с today и авто-линкуется; Reflect — daily-first + capture хоткеем + «On this day»; Obsidian Periodic Notes + Calendar; Things — Today/Upcoming поверх due-дат. Перенять у Logseq: захват по умолчанию в today, оттуда растаскивается ссылками.


### Слой задач поверх markdown-чекбоксов: кликабельные, write-back, индекс задач, LLM-extraction

**Обоснование:** Связка «мысль ↔ действие» отличает второй мозг от архива. Чекбоксы - [ ] рендерятся статично (не кликабельны, нет write-back ни в preview, ни в source); единственный таск-примитив — read-only #goal+frontmatter progress без статуса/дедлайна/действий (due парсится в индекс, но goals его не читает). Без задач планирование уходит в Things/Calendar и заметки отрываются от выполнения. Цель: кликабельный чекбокс с записью в .md; индекс задач (парсер - [ ] + опц. due/@tag) → панель Tasks (open/today/overdue); LLM task-extraction выносит «проверить до пятницы» в чекбокс; цели развить в мини-OKR с авто-прогрессом из чеклистов и видимым дедлайном. Всё поверх обычного markdown, без отдельного формата.

**Эталон:** Obsidian Tasks — query-синтаксис над markdown-чекбоксами (due/recurring/priority), задачи остаются обычным markdown но становятся запрашиваемыми; Tana — задачи как структурированные узлы со связью с заметками; Logseq TODO/DOING/DONE в outline; Things — авто-прогресс проекта из задач + подсветка дедлайнов.


### Obsidian-grade Live Preview на CM6 — форматирование при наборе, исчезающая разметка

**Обоснование:** Сейчас либо сырой markdown (## и **), либо read-only preview — печатать в отформатированном виде нельзя, постоянные прыжки source↔preview. Это, вероятно, фича №1 по влиянию на ощущение «продуманности до мелочей» и на дневной UX письма. Декорации CM6 скрывают маркеры разметки вне курсорной строки (жирный/курсив/заголовки/код/цитаты рендерятся inline, символы видны только на активной строке). Wikilink/тег уже декорируются — расширить на полную типографику; чекбоксы кликабельны; естественно решает интерактивные чекбоксы и embed ![[заметка]] (трансклюзия для MOC/дашбордов). XL, но определяет ежедневное ощущение продукта. Параллельно подтянуть: slash-команды (/) для вставки блоков, callouts, подсветку кода, frontmatter как properties-панель, unlinked mentions + hover-preview беклинков.

**Эталон:** Obsidian Live Preview (CM6, точно паттерн раскрытия разметки на активной строке) + embeds/transclusion ![[note]], ![[note#heading]]; Typora/Bear как «исчезающий markdown»; Notion/Craft — блочный WYSIWYG; Notion slash-меню как эталон.


### Безопасный фоновый sync + единый omnibox «найди ИЛИ спроси» как ежедневный вход

**Обоснование:** Два усилителя ежедневности. (1) Sync полностью ручной (две кнопки в SyncPanel) — бэкап, который надо запускать руками, не запускают. Планировщик уже умеет recurring/backoff/dead-видимость: добавить опциональный sync-kind (авто-коммит по тишине + периодический pull→push), но строго commit-then-pull с флашем буферов (см. баг force-checkout). (2) Концепт-док описывает единый элемент «/ — поиск, ? — вопрос к LLM», но это разнесено по палитре/сайдбару/чату. Свести в omnibox (расширить палитру): по умолчанию файлы+команды, ?/Tab → RAG-ответ инлайн с цитатами, показывать контентные сниппеты. Один хоткей открывает всё — сильнейший хабит-формирующий паттерн «дом, откуда начинается любое действие».

**Эталон:** Obsidian Sync — непрерывный фоновый near-realtime; Obsidian Git — авто-коммит по интервалу + push; Raycast/Spotlight — omnibar как единая точка входа с мгновенным режим-свичем по префиксу/Tab; Mem — «спроси или найди» в одном поле.


## Адверсариальный критик полноты


**Непокрытые ежедневные сценарии:**

- ВЕЧЕРНИЙ РАЗБОР INBOX (processing/triage). Карта добавляет захват в Inbox.md (append+таймстамп), но НЕ закрывает цикл: нет момента «открыл накопленное и разнёс по местам». GTD/PARA-ритуал «обработать входящие до нуля» отсутствует как концепция — нет вида «необработанное», нет жеста «эту строку → в новую заметку / в daily / в задачу / удалить». Без этого Inbox.md за 2 недели превращается в свалку, которую страшно открыть, и захват умирает с ДРУГОГО конца, чем диагностирует карта (не «лень записать», а «некуда девать записанное»). Reflect/Logseq: захват в today → растаскивание ссылками; Tana: inbox-node со swipe-обработкой. Это отдельная high-impact ставка, которой в карте нет.
- ПОИСК ПО СМУТНОМУ ВОСПОМИНАНИЮ («та заметка про X, писал кажется весной, что-то про Y»). Карта чинит ТЕКСТОВЫЙ возврат (подключить searchContent), но не temporal+semantic-возврат: нет фильтра по диапазону дат, нет «искать в заметках за период», нет сортировки результатов по времени. Для второго мозга, живущего годами, «когда» — второй по силе якорь памяти после «о чём». search_content имеет префильтр по папке/тегу, но не по дате. Это слепое пятно поверх и без того нерешённого created_at.
- ПОДХВАТ НЕЗАКОНЧЕННОЙ МЫСЛИ ПОСЛЕ ПЕРЕРЫВА («на чём я вчера остановился»). На Home есть «Продолжить» (по files.updated_at), но это «последний ТРОНУТЫЙ файл», а не «незавершённая мысль». Нет концепции draft/WIP/«додумать», нет якоря на месте, где остановился (последняя позиция курсора/скролл не сохраняются — openFile всегда anchor=0). Ежедневный второй мозг — это в основном ВОЗВРАТ к недоформулированному, а не создание нового. Карта это не выделяет.
- РАБОТА С ИЗОБРАЖЕНИЯМИ/СКРИНШОТАМИ КАК ЕЖЕДНЕВНЫЙ МАТЕРИАЛ. Карта ловит два симптома (paste/drop нет; convertFileSrc не в preview), но не оформляет это как ЖУРНЕЙ: «скриншот → в заметку за 1 жест» — базовый ежедневный захват визуала (диаграммы, доски, фото доски/доски/чека). Сейчас путь невозможен целиком: нельзя вставить, нельзя увидеть вставленное. Для «второго мозга каждый день» визуальный захват часто важнее текстового, а он отсутствует end-to-end.
- ДОВЕРИТЕЛЬНОЕ МУЛЬТИУСТРОЙСТВО (тот же vault на двух машинах через git). Карта чинит отдельные баги (force-checkout, флаш буферов), но не journey целиком: нет фонового pull при фокусе окна, нет индикатора «есть изменения на удалёнке», нет авто-разрешения тривиальных конфликтов — то есть «открыл Nexus на втором ноуте и просто продолжил» не работает. Это ежедневный сценарий любого, у кого больше одной машины, и он держит человека на Obsidian Sync/iCloud.
- БЫСТРАЯ ОБРАБОТКА НОВОСТИ В ЗНАНИЕ («прочитал в ленте → сохранил мысль/выдержку в vault»). News Feed существует, но изолирован (карта это отмечает как «0 связи»), однако не как JOURNEY: нет жеста «эта новость → заметка/цитата/в today». Ежедневный ритм «читаю входящий поток → откладываю ценное в свой мозг» оборван; лента живёт 30 дней и стирается. Это ежедневная точка входа, которая сейчас — тупик.


**Пробелы/ошибки аудита (важно — поправки к карте):**

- ПЕРЕОЦЕНЕНО (и опасно): «Карточка На этот день (created_at уже есть в files)» как quick-win с мгновенным офлайн-SQL. Я проверил indexer/mod.rs:198-214 — created_at заполняется значением mtime (тот же `mtime` идёт и в created_at, и в updated_at) ТОЛЬКО при INSERT. На свежем git-clone/синке у ВСЕХ заметок mtime = время синка, значит created_at врёт ровно так же, как heatmap/streak, который карта честно пометила багом. Карта диагностировала корень («updated_at врёт при ребилде/синке»), но НЕ перенесла вывод на on-this-day, который на этом же поле построен. Итог: «годовщины» покажут «год назад» для заметок 5-летней давности. On-this-day требует СНАЧАЛА честной таблицы edit_events / реального birthtime — иначе это not S, и эффект отрицательный (подрыв доверия). Это центральная ошибка карты.
- НЕДООЦЕНЕНО: автосейв подан как «дешёвый S, фундамент доверия», но в связке с фактами из самой карты он ОПАСЕН без парных фиксов. CM6 undo умирает при закрытии файла (карта это знает), истории версий нет, write не атомарен, external-sync молча затирает dirty (Editor.tsx:126-140, подтверждено). Включить автосейв ДО atomic-write + .nexus/history + баннера внешнего изменения = автоматизировать потерю данных (debounce запишет полупустой буфер поверх диска, перетёртого извне). Правильный порядок (atomic-write S → external-change-guard → autosave → history) в карте есть по пунктам, но НЕ зафиксирован как обязательная последовательность; читатель возьмёт «дешёвый S» первым.
- ПРОПУЩЕНО: курсор/скролл-позиция не сохраняются между открытиями (openFile в workspace.ts всегда создаёт буфер без позиции; Editor.tsx при switching ставит anchor=0). Для ежедневного второго мозга это тихое, но постоянное трение: каждый возврат к длинной заметке — прокрутка с нуля. Карта ловит «нет фокуса при создании», но не «теряется место при возврате» — а возврат частотнее создания.
- ПРОПУЩЕНО/НЕДООЦЕНЕНО: память чата (chat_vectors, k=3/280симв) и связи названы, но карта не заметила, что НЕТ единой «модели пользователя» поверх vault — ни авто-резюме «кто я / над чем работаю / мои темы», ни персистентного профиля. Reflect/Mem продают именно это («ассистент, который меня знает»). Это и есть обещанный «второй мозг», а в Nexus ассистент каждый раз начинает с RAG-холодного старта. Карта кладёт это в один из 11 findings лейна AI (i:high,e:L), но не поднимает в темы/ставки — для daily-стикинесс это недооценка.
- ПРОПУЩЕНО: нет concept «processing state» заметки (inbox/draft/permanent), нет тегов/папок жизненного цикла из коробки — то есть нет встроенного PKM-метода (PARA/Zettelkasten/GTD). Карта добавляет примитивы (capture, daily, tasks), но не МЕТОД, который превращает их в систему. Лучшие (Tana с supertags, Logseq с workflow-статусами) дают мнение «как думать»; Nexus останется набором инструментов без ритуала.
- СЛЕГКА ПЕРЕОЦЕНЕН Live Preview как «фича №1 по ощущению продуманности». Он XL и определяет ощущение ПИСЬМА, но для daily-RETENTION (вернуться завтра) сильнее работают захват+возврат+сохранность. Карта сама ставит автосейв/capture/search выше — но в strategic_bets риторика про Live Preview («вероятно, фича №1») конкурирует за внимание с тем, что реально держит ежедневность. Риск расфокуса дорожной карты.


**Если делать ОДНУ вещь:**

ОДНА вещь: сделать НЕТЕРЯЕМОСТЬ мысли по-настоящему гарантированной — в строгом порядке atomic-write (vault.rs:629, S) → guard внешнего изменения dirty-буфера (Editor.tsx:126-140, M) → debounced-autosave + flush на blur/closeTab/onCloseRequested (S) → .nexus/history снапшоты (M). Почему именно это, а не захват/поиск (которые я тоже считаю критичными): второй мозг — это в первую очередь ДОВЕРИЕ. Пока правки живут в RAM до Ctrl-S, внешнее изменение молча затирает буфер, запись не атомарна, а истории версий нет (всё подтверждено в коде) — каждый день использования это лотерея на тихую потерю годами накопленного. Никакой quick-capture, on-this-day или Live Preview не создаст привычку «доверять системе каждый день», если хоть раз мысль исчезла. Это фундамент, на котором стоят ВСЕ остальные ежедневные сценарии; его обязательно делать единым связным срезом в указанном порядке (включить autosave первым, без atomic-write+history, — значит автоматизировать потерю). ВАЖНАЯ ПОПРАВКА К КАРТЕ: НЕ брать «На этот день» как обещанный быстрый S — created_at заполняется из mtime и врёт после синка; сначала нужна честная временная таблица (edit_events), иначе ресёрфейсинг подорвёт то самое доверие, которое мы чиним.

### Захват и ежедневный поток (Capture & daily flow)


**Текущее состояние:** Захват в Nexus минимален и почти весь спрятан на экране Home. Создание заметки: `vault.ts → createNote(dir='', {baseName='Untitled'})` всегда пишет в КОРЕНЬ vault файл `Untitled.md` (`Untitled 1.md`, …), с пустым содержимым; вызывается из 5 мест (команда `file.new`/⌘N в commands-core.ts, кнопки «+» в Sidebar/FileTree/GroupPane, `newNote()` в HomeView). После записи `openFile()` открывает буфер, но редактор (Editor.tsx) НЕ вызывает `view.focus()` при монтировании/смене файла — курсор не ставится, фокус остаётся вне поля. Daily note и Quick capture СУЩЕСТВУЮТ, но ТОЛЬКО как две кнопки в блоке «быстрые действия» на Home (HomeView.tsx:127-145): `dailyNote()` хардкодит `Daily/${YYYY-MM-DD}.md` с телом `# дата\n\n`; `quickThought()` открывает/создаёт `Inbox.md` с телом `# Inbox\n\n`. У них НЕТ ни команды в палитре, ни хоткея (useKeymap.ts резолвит только зарегистрированные в commands-core команды; daily/inbox там не зарегистрированы). Шаблонов (templates) нет вообще. Дефолтная папка/имя новой заметки не настраиваются (нет полей в prefs.ts). Нет команд rename/delete файла ни во фронте, ни в бэкенде (commands/vault.rs: только open/list/read/write/resolve/tags/mtime) — «Untitled» переименовать из UI нельзя, а welcome-заметка обещает «Удалите эту заметку в любой момент», хотя удаления из UI нет. Онбординг (Onboarding.tsx) учит vault→AI→индексацию, но НЕ показывает, как быстро записать мысль. RU-локаль для `qaDaily` не переведена (ru.json:215 = «Daily note»).

### Редактор и письмо


**Текущее состояние:** Редактор — CodeMirror 6 в строгом SOURCE-MODE (сырой markdown с syntax-highlight через @codemirror/lang-markdown + defaultHighlightStyle), БЕЗ live-preview. Чтение — отдельный read-only режим (MarkdownPreview на react-markdown + remark-gfm + кастомный remarkNexus), переключается mode-float пилюлей (BookOpen/PenLine) и ⌘E/toggleMode; глобальный reading-режим форсит preview. Два режима НЕ совмещены в одном представлении (нет Obsidian Live Preview / Notion-блоков). Editor.tsx держит один EditorView, смена файла — dispatch без пересоздания, колбэки через ref. extensions.ts даёт: подсветку `[[wikilink]]`/`#tag` декорациями, клик по wikilink→навигация, async-автокомплит имён заметок внутри `[[…` (бэкенд listNotes топ-50, #22), lineWrapping, читаемую ширину (--editor-max-width), saveKey (⌘S). Inline-LLM встроен глубоко: ghost-text у курсора (inlineGhost.ts, StateField+виджеты Ghost/Pending/Error), ⌘I «продолжить», плавающий тулбар по выделению (inlineToolbar.ts: Переписать/Сократить/Продолжить), Tab/Esc accept/reject, rAF-троттл стрима (stores/inline.ts), aria-live (InlineAria.tsx). BacklinksBar — входящие ссылки из SQLite, сворачиваемый, клик→источник; «unlinked mentions» НЕТ. MarkdownPreview CSP-безопасен (без rehype-raw, urlTransform режет js:/data:), стилизованы заголовки/списки/таск-чекбоксы(нерабочие)/код/цитаты/таблицы/wikilink-чипы/тег-пилюли. doc-meta строка (mtime/слова/время чтения) только в preview. ЧЕГО НЕТ (подтверждено по коду): slash-команд, шаблонов, callouts/выносок, математики (KaTeX) и Mermaid (отложены под CSP), embed `![[заметка]]` (трансклюзия), рендера вложенных vault-картинок в preview (convertFileSrc только в FileViewer, не в MarkdownPreview), обработчиков paste/drop изображений и copy-as-markdown из браузера, форматных горячих клавиш (нет ⌘B/⌘I-италик/⌘K-ссылка — ⌘I занят LLM), фолдинга/outline/TOC, frontmatter-обработки (YAML рендерится сырьём), интерактивных чекбоксов, авто-создания заметки по клику на несуществующий `[[wikilink]]` (openLink молча no-op если resolveNote==null).

### Поиск, навигация, возврат к знаниям


**Текущее состояние:** Есть две раздельные поисковые системы. (1) Метаданный поиск `search_vault` (search/mod.rs::search_notes) — substring LIKE по path/title/именам тегов, LIMIT 100. (2) Гибридный поиск по ТЕЛУ `search_content` (hybrid_search: вектор usearch + FTS5/BM25 + граф-ранг → RRF → dedup, со сниппетами, стоп-словами RU/EN, префильтром по папке/тегу) — мощный и хорошо протестирован, но в реальном UI НЕ ВЫЗЫВАЕТСЯ нигде: его единственные потребители — mock и RAG-чат. Глобальный кеймап (useKeymap.ts) срабатывает только на комбо с модификатором, резолв пользователь>плагин>ядро, ремап в localStorage. НЕТ: истории недавних файлов / quick-switcher по последним открытым, навигации назад/вперёд, fuzzy-поиска, подсветки совпадений в результатах, выделенного хоткея на ⌘O/поиск, поиска по телу из палитры/сайдбара, исходящих ссылок (outgoing) в UI, навигации по результатам с клавиатуры в сайдбаре.

**Текущее состояние:** HomeView.tsx — большой статичный дашборд из секций (приветствие по часу + hero-поиск ⌘K + «Продолжить» + 5 быстрых действий + карточки: Daily brief, Недавние, Активность(heatmap/серия/тренд), Мини-граф, Прогресс целей, Статистика, Stale radar, Open questions, Context drift). - **[friction] Начальный скан индекса блокирует готовность AI/поиска без видимой деградации на огромных vault** `impact=med effort=M`
  - _Доказательство:_ event_loop (indexer/events.rs:63-69) на старте делает полный scan_vault ПЕРЕД обработкой инкрементальных событий; vault:changed/index-progress шлются, но пока скан идёт, поиск/RAG/беклинки работают по неполному индексу. Скан конкурентный с прогрессом N/M (indexer/mod.rs:496-535) — это хорошо, но на vault в десятки тысяч .md (особенно force-реиндекс при смене embedding-модели, vault.rs:540-545 чистит ВСЕ chunks) первый запуск/первая индексация может быть долгой, а UI лишь показывает счётчик.
  - _Почему важно:_ Холодный старт и «после смены модели» — моменты, когда ежедневный пользователь ждёт. Если поиск/граф/AI кажутся «сломанными» (пустыми) во время индексации без явного объяснения, это читается как баг. Скорость и предсказуемость старта — часть надёжности.
  - _Рекомендация:_ Индексировать сначала недавно открытые/изменённые файлы (priority queue по mtime), чтобы актуальное стало искабельным первым. Явный баннер «Индексация vault: N/M, поиск частичный» вместо тихого счётчика. Для force-реиндекса — фоновая переэмбеддизация без обнуления старого FTS до готовности нового (избежать окна, где поиск пуст).
  - _Эталон:_ Obsidian индексирует прогрессивно и остаётся отзывчивым; Linear/Raycast — мгновенный локальный поиск с фоновым прогревом. Перенять mtime-приоритизацию и явный partial-state индикатор.

- **[polish] usearch (векторы) сохраняется per-event без дебаунса — лишние fsync при пакетных правках** `impact=low effort=S`
  - _Доказательство:_ indexer/events.rs:95-99 после КАЖДОГО инкрементального события зовёт persist_vectors() (save usearch на диск). Комментарий сам признаёт: «Дебаунс самого save — позже при росте индекса». Watcher дебаунсит fs-события (400 мс), но серия разных файлов (массовая вставка, git pull многих файлов) даст серию сохранений индекса.
  - _Почему важно:_ На большом vault частый сброс usearch-файла — это IO-стук и подтормаживание во время активной работы/синка. Отзывчивость при пакетных изменениях — часть ежедневного комфорта.
  - _Рекомендация:_ Дебаунсить persist_vectors() (например, флашить не чаще раза в N секунд или по N событий, плюс гарантированный флаш при простое и на закрытии vault). reconcile_vectors (#5.1) уже страхует от потери при краше между commit и save, так что более редкий save безопасен.
  - _Эталон:_ Стандартная практика write-coalescing для индексов на диске (как WAL-checkpoint у SQLite). Перенять интервальный/счётчиковый флаш.

- **[polish] Конфликт-резолвер показывает «∅» для бинарных/больших файлов и грузит весь текст в память без диффа** `impact=low effort=M`
  - _Доказательство:_ git/mod.rs:117-124,432-453 — ConflictFile несёт base/ours/theirs как полные String; бинарь представлен пустой строкой (комментарий :122). ConflictResolver.tsx:190,203 рендерит весь текст в <pre> и предлагает ours/theirs/both/manual целиком на файл — нет построчного диффа/хунков (хотя в UI есть лейбл «hunk», conflict.tsx:168-169, по факту это весь файл). На большом конфликтующем .md это стена текста.
  - _Почему важно:_ Разрешение конфликта — стрессовый момент, где легко потерять данные неверным выбором. Построчный дифф резко снижает риск и когнитивную нагрузку. Для синкаемого второго мозга качество резолвера = доверие к sync.
  - _Рекомендация:_ Показывать построчный 3-way дифф (выделять расходящиеся участки), а не файл целиком; для бинарных/крупных — явная метка «бинарный, выберите версию» вместо пустого «∅». Можно считать дифф на фронте из base/ours/theirs.
  - _Эталон:_ VS Code 3-way merge editor, Obsidian Sync version-diff — построчное выделение конфликтов. Перенять хунк-уровневый дифф.

### Стратегические пробелы vs лучшие приложения мира


**Текущее состояние:** Nexus сегодня — это сильный «AI-ридер/мыслитель» поверх vault: индексация + гибридный RAG + реранкер, чат (vault/general/web с цитатами), inline-LLM в редакторе (continue/rewrite/summarize), беклинки, граф с тегами, Home-дашборд (статистика, недавние, daily brief, stale radar, open questions/context drift, активность-heatmap, прогресс целей), поиск противоречий, News Feed, чат-сессии с памятью, git-sync, плагины. Команд в lib.rs ~70 (egress/vault/graph/search/chat/inline/news/suggest/goals/home/digest/contradictions/scheduler/settings/plugin/git) — задач/todo/daily/template/calendar/capture/canvas/review СРЕДИ НИХ НЕТ. Создание заметки (`file.new` → `createNote`) делает пустой `Untitled.md` без шаблона и без даты. Чекбоксы `- [ ]` рендерятся в MarkdownPreview через remark-gfm, но статично: НЕ кликабельны, write-back в файл нет, агрегации задач по vault нет. «Цели» (goals/mod.rs) — единственный таск-подобный примитив: read-only список заметок с инлайн-тегом `#goal` + frontmatter `progress`, без статуса/дедлайна/действий. Temporal-слой есть только как stale-radar (старые заметки) и activity-heatmap (changes_today, серия дней) — нет journal/daily, нет on-this-day, нет date-индекса заметок. Концепт-док `docs/design/PKM_Home_Concepts.md` ОПИСЫВАЕТ daily note, быстрый захват мысли, умные шаблоны (классификатор+ghost), task extraction, habit tracker, сократические вопросы, авто-теги, маршрут по теме — но всё это либо в backlog-секции концепта, либо в `BACKLOG.md` как «vision, не брать автономно» (нужна сессия vision→AC). Реально не реализовано: задачи/todo, daily notes, quick capture/inbox, шаблоны, календарь/planner, canvas/доска, spaced repetition, мобайл, веб-клиппер, экспорт/публикация (есть только print→PDF одной заметки, `lib/print.ts`).


- **[gap] Нет быстрого захвата (quick capture / inbox) — главный разрыв ежедневного использования** `impact=high effort=M`
  - _Доказательство:_ В lib.rs нет команды захвата; `file.new` (commands-core.ts:51) вызывает `createNote` (stores/vault.ts:84), который создаёт пустой `Untitled.md` в текущем каталоге и открывает его в редакторе — это «создать файл», а не «бросить мысль и забыть». Нет глобального хоткея/окна захвата, нет папки Inbox, нет захвата в daily note. PKM_Home_Concepts.md:47 описывает кнопку «захватить мысль» в быстрых действиях Home, но она не реализована.
  - _Почему важно:_ Второй мозг живёт или умирает на трении захвата: если занести мысль дольше 2 секунд — пользователь не заносит, и база перестаёт быть полной. Это причина №1, по которой PKM забрасывают. Без капчура Nexus остаётся инструментом, который открываешь осознанно, а не рефлекторно каждый день.
  - _Рекомендация:_ Сделать глобальный quick-capture: системный хоткей (Tauri global-shortcut) открывает минимальное окно ввода → текст падает в `Inbox/` либо в конец сегодняшней daily note, окно закрывается. Плюс палитра-команда «Quick capture» и кнопка на Home. Без выбора файла, без переключения контекста. Позже — LLM авто-тег/авто-папка (концепт «авто-теги» уже описан).
  - _Эталон:_ Reflect и Mem — глобальный hotkey-капчур в today/inbox; Apple Notes — Quick Note (Fn-Q) по всей системе; Things — Quick Entry (Ctrl-Space) как эталон скорости. Перенять: один хоткей, окно поверх всего, Enter сохраняет и закрывает.

- **[gap] Нет ежедневных заметок (daily notes) — отсутствует якорь дня и временная ось мышления** `impact=high effort=M`
  - _Доказательство:_ Date-индекса заметок нет: temporal-слой существует только как stale-radar (home/stale.rs — старые заметки) и activity-heatmap (home/activity.rs: changes_today, серия дней по files.updated_at). Нет команды/шаблона daily note, нет `Journal/2026-06-13.md`-конвенции, нет навигации «вчера/сегодня/завтра», нет on-this-day. PKM_Home_Concepts.md:47 упоминает «открыть daily note» в быстрых действиях, но не реализовано.
  - _Почему важно:_ Daily note — это вход в систему КАЖДЫЙ день: место для лога, задач дня, быстрых мыслей, ретро. Это превращает «папку файлов» в дневник второго мозга и даёт LLM сильнейший временной контекст («что я думал в этот период»). Без него у daily brief нет куда «приземляться», а у пользователя нет ежедневного ритуала.
  - _Рекомендация:_ Ввести daily notes: команда/хоткей «Open today» создаёт/открывает `Journal/YYYY-MM-DD.md` по шаблону (см. находку про шаблоны); навигация prev/next день; виджет «Сегодня» на Home; quick-capture по умолчанию аппендит в сегодняшний день. Дата-поле в индексе → разблокирует календарь и on-this-day.
  - _Эталон:_ Roam/Logseq — daily note как центр всей системы (всё начинается со «today»); Obsidian Daily Notes core-plugin + Calendar; Reflect — daily-first дизайн. Перенять у Logseq: capture по умолчанию летит в today, оттуда растаскивается ссылками.

- **[gap] Нет задач/todo как первоклассной сущности — чекбоксы мёртвые, агрегации нет** `impact=high effort=L`
  - _Доказательство:_ Чекбоксы `- [ ]`/`- [x]` рендерятся в MarkdownPreview.tsx (remark-gfm, строки 77, тест MarkdownPreview.test.tsx:16) как статичные `<input type=checkbox>` — НЕ кликабельны, write-back в .md нет, в source-mode CM6 их тоже не тоглишь мышью. Нет команды/индекса для сбора задач по vault. Единственный таск-подобный примитив — goals/mod.rs: read-only список заметок с тегом `#goal` + frontmatter `progress`, без статуса done/дедлайна/приоритета/действий. Task extraction описан только в концепте (PKM_Home_Concepts.md:198) и BACKLOG как vision.
  - _Почему важно:_ Связка «мысль ↔ действие» — то, что отличает второй мозг от архива. Если задачи живут в отдельном приложении, заметки и дела расходятся, и Nexus не используют для планирования дня. Кликабельный чекбокс + сводка «все открытые задачи vault» — это причина возвращаться ежедневно, а не только когда «надо что-то записать».
  - _Рекомендация:_ Слой задач поверх markdown-чекбоксов (никакого отдельного формата): 1) кликабельный чекбокс в preview/live-preview с write-back в файл; 2) индекс задач (парсер собирает `- [ ]` + опц. `📅 due`/`@tag`) в таблицу → команда/панель «Tasks» с фильтром open/today/overdue; 3) LLM task-extraction из свободного текста (концепт уже есть) выносит «проверить до пятницы» в чекбокс. Цели (`#goal`) дополнить статусом.
  - _Эталон:_ Tana — задачи как структурированные узлы со связью с заметками (эталон «мысль=задача»); Obsidian Tasks plugin — query-синтаксис над markdown-чекбоксами (due/recurring/priority); Logseq — TODO/DOING/DONE прямо в outline. Перенять у Obsidian Tasks: задачи остаются обычным markdown, но становятся запрашиваемыми.

- **[gap] Нет шаблонов заметок — каждая новая заметка стартует с пустого Untitled.md** `impact=high effort=M`
  - _Доказательство:_ createNote (stores/vault.ts:84-91) пишет `opts.content ?? ''` — пустую заметку; baseName по умолчанию 'Untitled'. Команды/папки Templates нет в lib.rs. Frontmatter-парсер и frontmatter_fields есть (для goal/progress/due), но шаблонов поверх них нет. «Умные шаблоны» (классификатор→LLM→ghost/панель) детально спроектированы в PKM_Home_Concepts.md:125-160 и стоят в BACKLOG как vision-фича, но не реализованы; inline-LLM `/template` явно отложен (BACKLOG строка про IL-3b/IL-4).
  - _Почему важно:_ Шаблоны убирают трение и наводят структуру — встреча, литература, проект, ретро всегда заполняются одинаково, что делает заметки сравнимыми и запрашиваемыми (Dataview/goals). Для ежедневного второго мозга это ускоряет daily note и типовые записи; а «умный» вариант (LLM по похожим заметкам) — киллер-фича именно для AI-PKM-позиционирования Nexus.
  - _Рекомендация:_ Этап 1 (дёшево, разблокирует daily notes): статические шаблоны — папка `Templates/`, плейсхолдеры `{{date}}`/`{{title}}`/`{{time}}`, выбор при создании. Этап 2 (vision, отличает Nexus): «умный шаблон» из концепта — эвристика по названию/тегам → LLM с топ-3 похожих заметок → ghost-text, обучение на принятых/отклонённых.
  - _Эталон:_ Notion — templates с переменными как основа воркфлоу; Obsidian Templater — логика/дата/скрипты; Tana — supertags задают структуру автоматически. «Умный» уровень (LLM воспроизводит ТВОЙ паттерн) — то, чего нет ни у кого, и это естественная дифференциация Nexus.

- **[gap] Нет календаря/планнера — временное измерение не визуализируется и не планируется** `impact=high effort=L`
  - _Доказательство:_ Дата-навигации по заметкам нет (см. daily notes); единственное «время» — activity-heatmap (home/activity.rs, HEAT_DAYS) и stale-radar по age_days. Нет календарного вью, нет связи задач с датами (frontmatter `due` парсится в frontmatter_fields, но не используется ни одним вью — goals/mod.rs читает только `progress`), нет «что запланировано на сегодня/неделю».

Если в Nexus нельзя спросить «что сегодня/что просрочено», пользователь уходит планировать в Things/Calendar, и заметки отрываются от выполнения. Календарь поверх daily notes + due-дат замыкает цикл «думаю → планирую → делаю → ревью» внутри одного приложения.
  - _Рекомендация:_ После daily notes + due-дат задач: 1) мини-календарь в сайдбаре (как Obsidian Calendar) — клик по дню открывает/создаёт daily note, точки на днях с записями; 2) виджет «Сегодня/На неделе» на Home собирает задачи с `due` ≤ today. Использовать уже-парсящийся frontmatter `due`. Полноценный планнер (drag-задач по дням) — позже.
  - _Эталон:_ Things — «Today/Upcoming» как сердце ежедневного цикла; Reflect — календарь интегрирован с daily notes и встречами; Obsidian Calendar plugin — лёгкий мост дни↔daily notes. Перенять у Things модель Today/Upcoming/Anytime поверх due-дат.

- **[gap] Нет canvas/доски — невозможно мыслить пространственно и собирать кластеры визуально** `impact=med effort=XL`
  - _Доказательство:_ Компонентов canvas/whiteboard в apps/desktop/src/components нет (есть graph/GraphView — авто-силовой граф связей, но это не редактируемая доска: узлы нельзя свободно раскладывать в смысловые группы, добавлять карточки/стрелки/текст вручную). Граф — это read-only визуализация бэклинков, не творческая поверхность.
  - _Почему важно:_ Часть мышления — пространственная: разложить идеи, сгруппировать, провести стрелки, собрать борд по проекту/исследованию. Граф связей этого не даёт (он отражает ссылки, а не намерение). Для ежедневного второго мозга canvas — это место синтеза и планирования крупных вещей, которого сейчас нет совсем.
  - _Рекомендация:_ Средне-срочная ставка (не первый приоритет): canvas-вкладка воркспейса — бесконечный холст с карточками-заметками (embed существующих .md), свободными текст-нодами и стрелками, сохранение в `.canvas`-подобный JSON в vault. Реюз модели вкладок воркспейса. Начать с MVP: карточки + связи + перетаскивание.
  - _Эталон:_ Obsidian Canvas (формат .canvas, embed живых заметок на холст) и Heptabase (эталон визуального thinking-канваса со связью карточек и заметок) — главные референсы; tldraw как движок. Перенять у Heptabase идею «карточка на холсте = живая заметка», а не статичный снимок.

- **[gap] Нет интервального повторения (spaced repetition) — знание накапливается, но не закрепляется** `impact=med effort=L`
  - _Доказательство:_ В lib.rs/командах нет review/flashcard/SR; парсер не выделяет review-блоки (`==highlight==`, `Q::A`); планировщик (scheduler/mod.rs) умеет recurring-джобы, но review-очереди заметок/карточек нет. Не упомянуто даже в концепте.
  - _Почему важно:_ Второй мозг должен не только хранить, но и возвращать к тебе знание, чтобы оно не забывалось — это превращает архив в обучение. SR даёт сильный ежедневный ритуал («сегодня к повторению N карточек»), который удерживает пользователя в приложении даже в дни без новых заметок. У Nexus уже есть stale-radar (возврат к старым заметкам) — SR это его осмысленное развитие.
  - _Рекомендация:_ Лёгкая ставка поверх существующего: SR на основе markdown — выделять `==важное==` или строки `Q :: A` как карточки, простой SM-2/FSRS-планировщик, ежедневная очередь повторения как Home-виджет/команда. Состояние карточек — в .nexus БД (как stale_cache). Не делать отдельный формат — поверх заметок.
  - _Эталон:_ RemNote — заметки и флешкарты — одно и то же (эталон интеграции); Obsidian Spaced Repetition plugin — карточки прямо в markdown (`==cloze==`, `Q::A`); Mochi. Перенять у RemNote принцип «пишешь заметку — попутно создаёшь повторяемое знание, без отдельного шага».

- **[gap] Нет сквозного захвата с мобильного/из браузера — vault пополняется только за столом** `impact=med effort=L`
  - _Доказательство:_ BACKLOG явно откладывает мобайл: «Скелет apps/mobile/» и «nexus-md-parser как отдельный crate» — оба в Фазе 3/позже (строки 124-125), решение владельца «мобилка — отдельная веха после desktop-beta». Веб-клиппера тоже нет (egress/web-агент есть только для чтения новостей/поиска, не для сохранения страницы в vault). Захват возможен только в десктоп-приложении за компьютером.
  - _Почему важно:_ Мысли и интересные ссылки приходят не за столом. Если их некуда положить с телефона/из браузера, они теряются, и «второй мозг, которым пользуются каждый день» имеет дыру в покрытии ~половины дня. Это не обязательно полноценная мобилка — достаточно лёгкого канала захвата в тот же vault.
  - _Рекомендация:_ Не строить мобайл целиком сейчас. Дешёвый сквозной захват: 1) веб-клиппер — браузерное расширение/букмарклет → POST в локальный/синкаемый Inbox (или через git-sync репозиторий); 2) «отправить себе» через любой синк (vault уже в git). Полноценную мобилку оставить вехой, но дать канал входящих раньше неё. Учесть untrusted-injection для клипнутого контента (фундамент уже заложен в net.md).
  - _Эталон:_ Obsidian Web Clipper + мобильное приложение с share-sheet (эталон «сохранить в vault откуда угодно»); Notion Web Clipper; Reflect мобайл с быстрым капчуром. Перенять у Obsidian Web Clipper: markdown-извлечение статьи + выбор папки, всё в локальный vault.

- **[friction] Цели read-only без действий и без жизненного цикла — рядом с задачами, но не доведено** `impact=med effort=M`
  - _Доказательство:_ goals/mod.rs возвращает Vec<Goal>{path,title,progress} — чистый SQL-read по тегу `#goal` + frontmatter `progress`. Нет статуса (active/done/paused), дедлайна (frontmatter `due` парсится в индекс, но goals его не читает), приоритета, действий из UI (отметить done, обновить прогресс — только ручная правка frontmatter в файле). list_goals — единственная goals-команда в lib.rs.
  - _Почему важно:_ Цели — естественный мост между «вечными вопросами» и ежедневными задачами. Сейчас это пассивный список процентов, который надо обновлять руками в YAML — высокое трение, поэтому им не будут пользоваться ежедневно. Связав цели с задачами и дедлайнами, Nexus получит слой «к чему я иду», который осмысляет daily-работу.
  - _Рекомендация:_ Развить goals в мини-OKR: читать `due`/`status` из уже-индексируемого frontmatter; на Home-виджете целей — прогресс-бар + дедлайн + связанные задачи (чекбоксы из заметки-цели); действие «обновить прогресс/закрыть» пишет frontmatter из UI без открытия файла. Связать с слоем задач (находка про todo).
  - _Эталон:_ Tana — цели/проекты как узлы с подзадачами и датами; Notion — связанные базы goals↔tasks; Things — проекты с прогрессом. Перенять у Tana связь «цель → её задачи → прогресс считается автоматически».

- **[enhancement] Унифицированная строка «искать / спросить» из концепта не собрана в единый ежедневный вход** `impact=med effort=M`
  - _Доказательство:_ Концепт (PKM_Home_Concepts.md, строка про строку поиска) описывает единый элемент: префикс `/` — поиск по vault, `?` — вопрос к LLM. По факту это разнесено: палитра (mod+p, файлы+команды), поиск в сайдбаре (метаданные; контент-сниппеты в UI вообще не выводятся — BACKLOG про search snippet), и чат (mod+j). Единой точки «начни печатать — найди ИЛИ спроси» нет.
  - _Почему важно:_ Один вход снижает когнитивную нагрузку «куда мне идти за этим» и делает приложение мгновенно-полезным каждый раз при открытии. Это то, что у Raycast/Spotlight ощущается как «дом» — место, откуда начинается любое действие. Для ежедневного второго мозга единый omni-bar — сильнейший хабит-формирующий паттерн.
  - _Рекомендация:_ Свести в один omnibox (расширить палитру): по умолчанию — файлы+команды; `?`/Tab → режим вопроса к LLM с RAG-ответом инлайн (с цитатами, как chat web-режим); показывать контент-сниппеты в выдаче (BACKLOG отмечает, что их сейчас нет). Один хоткей открывает всё.
  - _Эталон:_ Raycast и Spotlight — omnibar как единая точка входа; Mem — «спроси или найди» в одном поле; Notion Q&A в поиске. Перенять у Raycast мгновенный режим-свич (символ-префикс/Tab) без отдельных окон.

- **[gap] Нет экспорта/публикации за пределы print-PDF — мысли заперты в приложении** `impact=low effort=M`
  - _Доказательство:_ Экспорт ограничен `file.print` → `lib/print.ts` (печать/PDF одной активной заметки). Команд экспорта набора заметок, публикации (share-ссылка/статический сайт), или экспорта в HTML/Markdown-бандл в lib.rs нет. Vault — это .md-файлы (хорошо для портабельности), но «поделиться результатом мышления» наружу пути нет.
  - _Почему важно:_ Второй мозг ценен и тем, что из него рождается выход — статья, заметка коллеге, опубликованная мысль. Если делиться можно только PDF одной страницы, Nexus остаётся приватным write-only хранилищем. Лёгкая публикация/экспорт замыкает цикл «потребляю → думаю → произвожу → делюсь», увеличивая ежедневную ценность.

Публикация (Obsidian-Publish-стиль статический сайт) — отдельная veha, учесть egress/CSP. Достаточно начать с «copy as clean markdown» (резолв ссылок/embed для вставки в другое приложение).
  - _Эталон:_ Obsidian Publish — vault → статический сайт; Notion — публичные share-ссылки; Craft — красивый экспорт/публикация документа. Перенять у Craft качество одно-кликового «поделиться этим документом как ссылкой/PDF/markdown».
