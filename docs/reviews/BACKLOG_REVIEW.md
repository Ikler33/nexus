## 1. TL;DR

| Тяжесть | Кол-во |
|---|---|
| **blocker** | 6 |

Главные риски (одной фразой):

1. **Eval-гейт качества RAG выключен в CI** — главный merge-gate проекта (AC-EVAL-3/AC-Q-4) держится на ручной дисциплине; автономный агент не может доверять зелёному CI и не имеет offline-сигнала, чтобы трогать ранжирование. **Это корневой блокер — разблокирует ~6 пунктов `🔬`-блока.**
2. **Вся секция «Идеи/vision» (~60% строк бэклога) не имеет ни одного AC** и несёт зашитые продуктовые решения владельца — автономный агент физически не закроет эти пункты «от и до» и сожжёт лимиты впустую.
3. **Три блокирующих security-gate (AC-SEC-7 prompt-injection, AC-SEC-6 Redacted-логи, AC-SEC-4 для core-эндпоинтов) не реализованы и НЕ помечены отложенными** — прямое нарушение собственного правила «no silent caps»; CI вообще не содержит security-job.
4. **Vision-фичи стоят раньше своего фундамента**: 3 неспроектированные подсистемы (планировщик фоновых задач, парсинг frontmatter, core-egress в интернет) — фичи перечислены так, будто инфраструктура есть; последняя прямо конфликтует с приватной моделью ADR-002/005.
5. **Бэклог рассинхронен с реальностью**: Фазы 2–4 в `CHANGELOG` закрыты, а BACKLOG держит «Фаза 1 (текущая)» и очередь как устаревшую → крон-сессия рискует переделывать готовое или встать на пустую очередь.

### 2.2. Внешние зависимости / секреты / ручные шаги — ПОДГОТОВИТЬ ЗАРАНЕЕ

Серверы и инфраструктура:
- **Эмбеддинг-серверы `:8081/:8082/:8083`** «слетают после ребута мака» — почти все `🔬`/«живые» пункты упрутся в недоступность. Корень снимается фикстурой A1, но для калибровки/реранкера сервера всё равно нужны.
- **Reranker-сервер** с эндпоинтом `/rerank` (сейчас `:8082` = jina-эмбеддер кода) — выбрать модель (bge-reranker-v2-m3?), поднять, проверить контракт (A6).
- **SearXNG на VPS** — для web-агента (A8); в репо нет.
- **Backend-эндпоинт приёма crash-репортов** на VPS (A13); в репо нет.
- **Реестр/хостинг плагинов** (артефакты + привязка id→издатель) для marketplace (A4).

Секреты и ключи:
- **Tauri-keypair** для подписи манифеста обновлений — приватный ключ владельца, в CI-secrets (A12). Утечка = подмена обновлений у всех.
- **Cloud API-ключ** (OpenAI/Anthropic) для cloud-fallback — **в OS keychain** (расширить `creds.rs` новым service `nexus-llm`), НЕ в `local.json`; выставить права 0600 на `local.json`. Заявлено в §11, не реализовано.
- **Ключ издателя плагинов** для подписи артефактов marketplace (A4).

Аккаунты и ручные процедуры (неавтоматизируемо):
- **Apple Developer** ($99/год) + Developer ID в Keychain + интерактивный `notarytool` (Apple ID) — Apple-нотаризация (A12).
- **Windows Authenticode-сертификат** (A12).
- **Физический девайс/симулятор iOS** (Mac + Xcode) для мобильного трека (A14).
- **Push в remote** — настройка GitHub-классификатора + workflow-scope токена для кросс-ОС CI и merge (A17).
- **Артефакт диагностики бага графа** от владельца: терминальный вывод паники + 3 ответа (A9).
- **Личный vault владельца** (`NEXUS_TEST_VAULT`) ИЛИ новый golden-кейс — для воспроизведения стоп-слов (A15).

Продуктовые решения владельца (нужны до старта vision-фич — фиксировать как AC):
- Стратегия памяти агента (что помнить, политика вытеснения LRU/важность/TTL, бюджет, формат инжекта) + golden.
- Профиль интересов + ключевые слова + список RSS-источников + egress-policy для News Feed.

### HIGH

**Design-gap: отсутствующий фундамент vision** `[консенсус: Архитектор + Разработчик + QA — разные формулировки одной дыры]`
- **H1. Планировщик фоновых задач отсутствует** [Архитектор, подтверждает Разработчик]. ≥5 vision-фич + весь Home (News Feed scheduled раз/сутки, Карта компетенций раз/мес, Поиск противоречий кэш, on-change дебаунс ~30с, persistent-кэш с инвалидацией по mtime) требуют планировщика. В ARCHITECTURE 0 упоминаний schedul/cron/job-queue; §5.1 — только reconcile-очередь индексации (сама `✂️`-отложена). **Правка:** завести фундаментальный пункт «Планировщик фоновых LLM/IO-джобов» (persistent-очередь + триггеры on-open/on-change/scheduled/manual + кэш-таблица с mtime-инвалидацией), внести в ARCHITECTURE как подсистему §4-уровня + ADR (tokio-интервалы-пока-открыто vs OS-cron). Триггер для всех scheduled/cached фич.
- **H2. Парсинг frontmatter в структурированные поля отсутствует** [Архитектор, подтверждает QA]. `progress/draft/wip/due/evergreen/goal` читаются как структурированные поля, но `parser/mod.rs:47-48` + `indexer/mod.rs:180` хранят frontmatter как СЫРУЮ YAML-строку (нет `serde_yaml` в Cargo.toml); таблица `aliases` есть в схеме, но НИКОГДА не заполняется. ARCHITECTURE §5 комментирует «JSON blob», фактически raw-YAML. **Правка:** фундаментальный пункт «Парсинг frontmatter (YAML→JSON + типизированный доступ)» — триггер для: Прогресс целей, Stale radar, Умные шаблоны, Dataview, резолв aliases. Дёшево, широко разблокирует. Автономно-безопасно (чистый Rust + тесты).
- **H3. core-egress в интернет конфликтует с приватной моделью** `[консенсус: Архитектор + Безопасность]`. web-агент/SearXNG для CHAT-ядра — новая граница приватности, не покрытая ADR; «как cloud-fallback ADR-005» — некорректная аналогия (ADR-005 = chat-LLM, не произвольный веб-поиск с tool-use). Конфликтует с `local_only` (`permission.rs:112`) + §6.2 vault-grounding. → A8.

**Sequencing / фундамент раньше потребителя** `[консенсус: Архитектор + Разработчик + QA]`

## 6. Краткие резюме по линзам

- **Архитектор (BACKLOG ↔ ARCHITECTURE/ADR).** Бэклог дисциплинирован по «no silent caps» и согласован с ADR в закрытых фазах, но вся секция vision стоит на ТРЁХ неспроектированных подсистемах (планировщик, frontmatter-parse, core-egress), не вскрытых как блокеры; ядро рантайма плагинов отложено, из-за чего marketplace/SDK висят на незакрытом фундаменте.
- **Разработчик-реализатор.** Большинство отложенных пунктов имеют корректный триггер, но как руководство к коду бэклог содержит ловушки: фактическая раскладка репо (packages/plugins пусты, mobile нет, 2 миграции) противоречит §2/§5; NEEDS-DECISION-пункты (реранкер, токенайзер, suggest-2) не помечены флагом в самом бэклоге; ряд vision-фич и editor-extensions недооценены.
- **QA / Тестировщик.** Реализованные срезы Ф0–3 в массе тестируемы, но три пласта «готово» определены нетестируемо: вся vision-секция без DoD, баг графа без репро/гейта, и блокирующие security/perf-гейты (AC-SEC-7, AC-Б10-4, AC-Б2-2, AC-PERF-1) не закрыты автотестом и не помечены отложенными; главный eval-гейт RAG живёт только локально — регрессия в PR проходит незамеченной.
- **Аудитор автономности.** Бэклог реалистично метит то, что нельзя делать вслепую, но содержит системные дыры для автономного агента: ~60% (vision) без AC и с зашитыми решениями владельца; инфраструктура автономии хрупкая (серверы «слетают», нет offline-мока метрик); класс пунктов требует визуальной/физической проверки; очередь NIGHT-PLAN исчерпана и устарела относительно CHANGELOG → крон жжёт лимиты вхолостую.
- **Безопасность / приватность.** Каркас сильный (capability-broker, anti-traversal, net-allowlist, secret-scan, keychain), но есть незафиксированные дыры: CI без security-job; AC-SEC-7 (injection) и AC-SEC-6 (Redacted) не реализованы и не отложены; egress-контроль для core описан как «opt-in» без механизма; SSRF-валидация `*.url` покрывает только плагины; cloud-ключ-в-keychain и подпись auto-updater заявлены, но не реализованы — внести гейтами в CI до фаз egress/релиза.
- **Продукт / консистентность.** BACKLOG дисциплинирован на ~90%, но: кластер git-автоматизации (auto-commit/auto_pull/squash) декларирован в config.json/§8.2, реализован только ручной режим, и НЕ отслеживается (нарушение «no silent caps»); «Фаза 5» придумана для updater (фаз только 0–4); Home продублирован 3×; vision-critical «чат→агент» сидит в «Идеях», тогда как разрыв (чат всегда грунтуется в vault) активен уже сейчас.
