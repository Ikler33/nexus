## ADR: Канбан/задачи с AI — «задача = заметка со status», доска как вью над frontmatter

**Статус:** Принято (decision-complete, синтез 5 разведданных). **Дата:** 2026-06-16. **Следующая свободная миграция:** 018 (но в MVP не нужна — см. ниже).

### Контекст
Владелец заказал top-level раздел «Канбан/задачи» с тесной AI-интеграцией, Obsidian-паритетом Properties/тегов и разбивкой по проектам. Nexus — Tauri 2 + Rust бэкенд / React-TS фронт, локальный LLM/RAG, локально-первый, сохранность данных пользователя — мандат №4. Ключевые ограничения инфры (проверены по коду):
- **serde_yaml архивирован** (security-гейт владельца) → frontmatter читается ручным line-парсером (`parser/mod.rs`), сериализатора (write-back) НЕТ.
- `frontmatter_fields` (миграция 003) уже индексирует **плоские top-level скаляры** (без ведущих пробелов/`-`) → `status`/`project`/`priority`/`due` индексируются «бесплатно».
- `goals::list_goals` — готовый эталон кросс-файлового read по frontmatter (JOIN file_tags + LEFT JOIN frontmatter_fields), офлайн, детерминированный.
- Атомарная запись `atomic_write` (SAFE-1, tmp→fsync→rename), guard внешнего изменения (SAFE-3), снапшот истории (SAFE-5) — есть.
- ActivityBar + ui.ts (`openNews`/`openHome`) + App.tsx:246 — готовый паттерн top-level вью (~5 точек правки).
- AI: три тира (`chat`/`chat_fast`/`chat_util` Qwen3-4B :8084) + embedder bge-m3; JSON-batch-классификатор со строгим контрактом и `failed`-учётом (`news/llm.rs`); injection-fencing (`injection_marker`); eval-харнесс с baseline-гейтом (`eval/mod.rs`).

### Решение (форки разрешены однозначно)
1. **Задача = заметка с frontmatter-ключом `status`** (наличие ключа = задача). Источник: TaskNotes + goals-паттерн. Не папка, не обязательный тег.
2. **Доска = сохранённый вью над набором заметок** (Notion/Bases/Anytype-Set), колонка = значение `status`. Конфиг — `.nexus/boards/<slug>.json` (синк с vault), дефолты — `.nexus/config.json`.
3. **DnD = хирургическая правка ОДНОГО frontmatter-ключа** новой командой `set_frontmatter_field(path,key,value)->hash` через `atomic_write`, БЕЗ re-emit YAML (serde_yaml нельзя). Эта же команда пишет `project`/`priority`/`due` и питает Properties-панель — единая точка записи frontmatter.
4. **Ручной порядок — в board JSON** (`order:{колонка:[пути]}`), не во frontmatter (иначе фантомные `edit_events`, шум git, ресорт-дрейф — анти-паттерн Notion).
5. **Статусы — типизированные колонки per-board** с флагом `doneLike`, лейблы отдельно от raw-id (Linear-категории).
6. **Проекты = frontmatter `project:`** (плоский скаляр, уже в индексе); папка/тег — отклонены как первичная модель.
7. **Индексация — НЕ новая таблица:** переиспользовать `frontmatter_fields` + `file_tags`, выборка доски — клон `list_goals`. БД остаётся rebuildable-кэшем.
8. **Properties-паритет:** реестр типов по имени (`.nexus/property-types.json`, как Obsidian types.json) + эвристика вывода; 5 виджетов MVP (text/list/number/checkbox/date), datetime+спец-tags вторым срезом; invalid → жёлтое поле + escape в source (без авто-конверта).
9. **Теги:** Unicode/кириллица (is_alphabetic), автокомплит инлайн+frontmatter, тег-пейн с деревом/частотами.
10. **AI MVP:** A1 (заметка→задача, без LLM), A2 (застрявшие, SQL/edit_events), A3 (план-дня отбор), A4 (авто-тег closed-vocabulary). MVP+1: B1 (разбить на подзадачи), B3 (саммари) — под структурными/числовыми гардами. Vision без измеримого порога → вынесено в «требуют vision→AC».

### Рассмотренные альтернативы (отклонены)
- **Колонка ≠ статус (Trello/Kanban-плагин)** — теряет единую модель «задача=заметка», карточка-просто-текст не запрашивается из vault. Отклонено.
- **Порядок во frontmatter каждой задачи** — слетает при ресорте (анти-паттерн Notion), фантомные `edit_events`, шум git. Отклонено в пользу board JSON.
- **Отдельная таблица `tasks`** — второй источник правды, лишняя миграция; текущий индекс покрывает запрос. Отклонено (YAGNI, BACKLOG).
- **Папка/тег как модель проекта** — жёсткая привязка файла, ломает мульти-проект и rename. Отклонено в пользу `project:`-скаляра.
- **Введение YAML-либы для write-back** — нарушает security-гейт владельца. Отклонено в пользу хирургической правки строки.
- **Эмодзи-синтаксис задач (📅⏫🔁) как единственный формат** — нечитаемо, требует CSS-тем. Отклонено; метаданные в типизированный frontmatter (эмодзи — опциональный рендер).
- **Сетевой/облачный AI** — урок Height (закрылся) + локальность Nexus. Отклонено; всё локально под egress-гардом.

### Последствия
- **Плюс:** ноль новых миграций в MVP; единая модель метаданных всего vault (доска + Properties + теги пишут один writer); git/diff-friendly; переносимость доски; локальный AI без vision-блокеров в первом срезе.
- **Минус/риск:** write-back round-trip без YAML-либы — самая рискованная инженерная часть (митигация property-тестами идемпотентности и MalformedFrontmatter-отказом); Properties-панель — крупный новый UI (сверять с макетом владельца ДО постройки); кириллица-теги меняют зафиксированный тест.
- **Процесс:** после каждой крупной реализации — adversarial-review диффа перед мержем; vision-фичи B1/B3 — короткая AC-сессия владельца (пороги judge 3.5/5, faithfulness 0.9) перед мержем B-среза; на дизайн-переносе Properties — скриншот-сверка.

### Единственный новый бэкенд-примитив
`set_frontmatter_field` (хирургическая правка одного ключа) — закрывает DnD-смену статуса И Properties-панель (требование №5) одним writer'ом.
